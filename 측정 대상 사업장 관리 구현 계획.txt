측정 대상 사업장 관리

1. 계획 생성(기능 및 버튼)-->삭제

2. 측정년도, 측정주기, 지정지청, 사업장명 검색, 주소 검색, 실시여부 같은 줄에 검색, 업체 추가, 측정 대상 사업장 엑셀 업로드 버튼 유지

3. 측정 대상 사업장 목록과 같은 줄의 우측에 계획담당자(강종구, 이주형, 한기문) 목록상자(가운데 정렬), 엑셀 다운로드 버튼 배치

4. 측정 대상 사업장 목록 항목 : 실시여부, 코드, 국고결과, 계획담당자, 전회측정일, 전회측정 추기, 금회 예정일, 측정예정월, 금회측정확정일, 업종분류, 주소, 소재지 관할청, 미수횟수, 사업장명, 담당자명, 담당자 휴대폰, 회사전화번호, 비고, 관리

5. 기능구현
   1) 실시여부 : 실시, 미실시, 거래종료
   2) 미수횟수 : 측정일지 기준으로 조회하여 값 연동
   3) 측정 대상 사업장 엑셀 업로드 양식 제공(핈수값 필히 명시 : 추후 자료의 연동 고려)
      - 실시여부, 코드, 국고결과, 계획담당자, 전회측정일, 전회측정 추기, 금회 예정일, 측정예정월, 금회측정확정일, 업종분류, 주소, 소재지 관할청, 미수횟수, 사업장명, 담당자명, 담당자 휴대폰, 회사전화번호, 비고 중 연동되는 테이블의 measurement_journal 테이블의 항목 명칭을 함께 적어준다. 양식 1행의 필드에 적어준다.
      - 매칭(연동)되는 DB 테이블 : business_info(약어 : m.i), measurement_business(약어 m.b), measurement_journal(약어 m.j), national_support_application(약어 m.s.a)
        (예 : 1행 매칭(연동)되는 테이이블명의 약어 + 필드으로 코드의 경우 1행에 "m.i_code"처럼 표기한다.
   4) 항목별 매칭 우선 순위와 고려사항
      - 실시여부 : 실시(바탕색 : 연한 녹색), 미실시(바탕색 : 연한 노랑색), 거래종료(바탕색 : 연한 적색)
      - 코드 : m.i
      - 국고결과 : m.s.a
      - 계획담당자 : 한기문, 강종구, 이주형만 해당
      - 전회측정일 : m.j
      - 전회측정 추기 : m.j
      - 측정예정월 : : 전회측정일 기준으로 주기를 합산한 일자의 월(1월, 2월, 3월, ~, 12월 표기)
      - 금회측정확정일 : 사용자가 직접 입력
      - 업종분류 : 업로드 값(유효성 검사는 business_category name필드 참고)
      - 주소 : m.b
      - 소재지 관할청 : m.b 자료를기준으로 하나, UI는 약어로 표시(폴더 내의 노동지청별 관할지역.csv 파일 참고)
      - 미수횟수
      - 사업장명 : m.b
      - 담당자명 : m.b_manager_name & manager_position (예 : 홍길동 팀장)
      - 담당자 휴대폰 : m.b_manager_mobile
      - 회사전화번호 m.b_future_measurement_period
      - 비고 : 
   4) 사용자가 "업체 추가" 버튼을 클릭하여 기본 정보에 대한 개별 등록 입력할 수 있도록 구현
   5) 자료의 연동(매핑)은 최신 자료를 기준으로 하며, 최신자료(우선 순위)라 함은 예를 들어 2026년 상반기의 경우2026년 상반기 --> 2025년 하반기 --> 2025년 상반기 --> 2024년 하반기 --> 2024년 상반기로 5개 주기까지로 한다.

6. 사업자가 측정 대상 사업장 엑셀 업로드 시 작성된 업체가 누락없이 등록되어야 함.
