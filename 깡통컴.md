# 깡통 컴퓨터(개발 서버) 환경에서의 건강디딤돌(MES) 조회 가이드

본 문서는 환경 셋업이 되지 않은 깡통 컴퓨터(개발 서버)에서 건강디딤돌 자동 신청 및 수동 결과 조회 기능을 기동할 때 발생하는 오류와 해결 방법에 대해 정리한 가이드입니다.

---

## 1. 깡통 컴(개발 서버)에서 기동 시 오류 여부
**오류가 발생합니다.**

건강디딤돌 결과 조회 기능은 내부적으로 **파이썬(Python)과 셀레니움(Selenium)을 이용한 헤드리스 크롬 브라우저**를 띄워 공단 사이트에서 데이터를 크롤링해오는 방식을 사용합니다. 따라서 관련 환경이 셋업되어 있지 않은 깡통 컴퓨터에서는 정상 구동이 불가능합니다.

---

## 2. 주요 오류 원인 및 현상

### ⚠️ 예상되는 오류 원인
1. **Python 및 라이브러리 부재:** 
   - 서버 OS에 파이썬이 설치되어 있지 않거나, 크롤링 필수 라이브러리인 `selenium`, `webdriver-manager`가 설치되지 않은 경우 백그라운드 프로세스가 시작하자마자 비정상 종료됩니다.
2. **구글 크롬(Chrome) 브라우저 부재:**
   - GUI 브라우저가 없는 Linux 서버나 크롬 브라우저가 설치되지 않은 PC에서는 셀레니움이 브라우저 인스턴스를 띄우지 못해 `WebDriverException` 오류가 발생합니다.

### 🚫 오류 발생 시 화면 현상
- 웹 화면에서 [새로고침] 또는 자동 조회를 실행하면 잠시 **"신청중(뱅글뱅글)"** 상태를 유지하다가, 크롤러 오류를 감지한 뒤 **"현재 신청 상태: 실패"**로 최종 변경되며 실패 상세 로그가 기록됩니다.

---

## 3. 해결 및 환경 구축 방법

개발 서버(깡통 PC)에서도 건강디딤돌 조회가 정상 작동하게 하려면 아래 단계에 따라 환경을 구성해야 합니다.

### [Step 1] 구글 크롬 브라우저 설치
- 컴퓨터 환경에 맞는 공식 구글 크롬(Chrome) 브라우저를 다운로드하여 설치합니다.

### [Step 2] 파이썬(Python 3) 설치
- 공식 Python 배포판(3.8 버전 이상 권장)을 설치합니다.
- 설치 시 **"Add Python to PATH"** 옵션을 반드시 체크하여 시스템 어디서나 `python` 명령어를 실행할 수 있도록 설정해야 합니다.

### [Step 3] 크롤러 의존성 라이브러리 설치
- 명령 프롬프트(CMD) 또는 터미널을 열고 아래 명령어를 실행하여 필수 파이썬 라이브러리를 설치합니다.
```bash
pip install selenium webdriver-manager
```

---

## 4. 참고 사항 (사용자 로컬 PC 상태)
- **사용자님의 현재 컴퓨터(작업 중인 로컬 PC)**는 진단 결과 이미 `Python`, `selenium`, `webdriver_manager` 및 구글 크롬 브라우저가 **모두 정상 설치 및 셋업 완료된 상태**입니다.
- 따라서 사용자님의 컴퓨터에서는 추가 설정 없이 즉시 수동 동기화 및 자동 조회를 테스트하셔도 완벽하게 작동합니다.
