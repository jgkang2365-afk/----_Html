# 시스템 핵심 비즈니스 로직 가이드 (The Joo Rule)

이 문서는 측정일지 시스템의 데이터 무결성을 유지하고 회귀 오류를 방지하기 위한 핵심 비즈니스 규칙을 정리한 것입니다.

---

## 1. [The Joo Rule] 지청별 예외 발행일 규칙
특정 조건의 사업장에 대해 전자계산서 발행일을 자동으로 계산합니다.

- **대상 영역**: 지정지청이 **'대전지청'** 또는 **'천안지청'**인 경우
- **대상 업종**: 업종분류가 **'공업사'**인 경우
- **규칙**: 전자계산서 발행일(`electronic_invoice_date`)을 **측정일 익일(워킹데이 기준)**로 자동 기록
- **적용 시점**: 2026-04-11 이후 측정 건부터 적용 (소급 적용 금지)
- **워킹데이 산정**: 주말(토, 일) 및 한국 법정 공휴일을 제외한 다음 영업일

## 2. 데이터 동기화 정책 (Latest Wins)
엑셀 데이터와 시스템 데이터 간의 충돌 발생 시 처리 원칙입니다.

- **원칙**: 엑셀 원본(`측정사업장.xls`, `사업장정보.xls`)에 데이터가 존재할 경우, 시스템 내 기존 필드 값을 **무조건 덮어씀**
- **대상 필드**: 사업자번호, 산재번호, 개시번호, 근로자수, 이메일, 담당자 연락처 등
- **목적**: 전산망 데이터의 원본성을 시스템 전체에 유지하여 정보 파편화 방지

## 3. 멀티 데이트(Multi-Date) 측정 관리
불연속적인 다수 일자의 측정 건을 관리하는 규칙입니다.

- **측정일수 계산**: 시작일과 종료일 사이의 기간 중 **주말 및 공휴일을 제외한 실제 작업일(영업일)**만 카운트
- **공시료 코드**: 멀티 데이트 발생 시 각 일자별 공시료 코드를 "4/21: B, 4/22: C" 형식으로 병합 관리
- **측정자**: 여러 명의 측정자가 참여할 경우 콤마(,)로 구분하여 통합 기록

## 4. 연번 부여 알고리즘
- **5인 이상 사업장**: 연번을 1씩 증가시키며 고유 번호 부여
- **5인 미만 사업장**: 직전 사업장에 부여된 연번을 재사용(중복 허용)하여 행정 관리 일관성 유지

## 5. 국고지원 상태값 정규화
- **내부 표준값**: **'대상'**, **'비대상'** (지원, 미지원 등 변종 값 지양)
- **UI 표기**: 항목명은 **'국고지원'**을 유지하여 사용자 익숙함 보호
- **데이터 변환**: 엑셀 업로드 시 '지원', '지원대상' -> '대상'으로, '미지원' -> '비대상'으로 자동 정규화
- **전파 규칙**: `measurement_target_business`의 최신 국고 상태는 일지(`measurement_journal`) 생성 및 보정 시 항상 최우선으로 반영됨

## 6. 일련번호 자동 부여 제외 (기타매출) 처리 규칙
'일련번호 자동 부여 제외'를 선택한 측정 건을 일반 측정비와 분리하여 관리합니다.

- **분류 트리거**: `revenue_type === '기타매출'` (UI의 '일련번호 자동 부여 제외' 체크 시)
- **리스트 노출**: 매출관리 화면의 '측정비' 탭에서 제외하며, **'기타'** 탭의 매출 내역 리스트에 포함함
- **집계 원칙**:
    - 매출 집계 및 년도별 통계 테이블에서 지사별(대전, 천안 등) 측정비가 아닌 **'기타'** 카테고리로 합산함
    - 미수금 상세 내역 조회 시 '기타' 카테고리 내에 해당 일지들이 포함되어야 함
- **수정 로직**: 기타 매출 내역에서 일지 기반 항목(`source: "journal"`) 수정 시, 일반 기타 매출 모달이 아닌 **기존 측정일지 수정 폼**을 호출하여 데이터의 원본성을 유지함

## 7. 업종 분류(business_category) 결정 알고리즘
사업장의 업종을 결정할 때 데이터 정합성을 위해 다음의 우선순위 알고리즘을 따릅니다.

1.  **1순위 (기초 데이터)**: `measurement_target_business` 테이블의 `business_category` 값을 최우선으로 사용함.
2.  **2순위 (과거 데이터 추적)**: 1순위 값이 없을 경우, 해당 사업장의 **직전 주기** 데이터를 `measurement_journal` 테이블에서 조회함. (단, 현재 주기 기준 **최대 3주기 전**까지만 소급하여 유효한 데이터를 가져옴)
3.  **3순위 (수동 선택)**: 위 두 단계에서 데이터를 찾을 수 없는 경우에만 사용자가 UI의 **'선택'** 메뉴를 통해 직접 업종을 지정함.

## 8. 미수금 안내 문자 발송 및 자동 취합 규칙
개별 사업장의 미수금에 대해 수동으로 안내 문자를 구성 및 발송하기 위한 규칙입니다.

- **미수금 판정 공식**: `측정비(사업장) - [입금액(사업장) + 입금액2(사업장)] > 0` 인 모든 건을 미수금 대상으로 판정합니다. (입금일자 입력 여부와 무관하게 잔액이 남아있으면 미수로 취급)
- **복수 미수 건 자동 취합**: 동일 사업장명(`business_name`)을 기준으로 존재하는 모든 미수 건을 실시간으로 취합하여 한 통의 문자로 결합합니다.
  - 대상 주기 목록은 오래된 순서대로 정렬하여 중복 제거 후 `2025년 하반기와 2026년 상반기` 혹은 `2024년 하반기, 2025년 상반기, 2025년 하반기` 형태로 결합하여 본문에 명시합니다.
  - 미수금액은 해당 사업장의 모든 미수 건의 잔액을 합산한 `미수합계금액`으로 표기합니다.
- **안내 문자 템플릿 구조**:
  - 문자 내용은 `"안녕하십니까! 한결작업환경컨설팅입니다. {periodsText} 작업환경측정 수수료 미수금 {formatAmt}원 이오니 확인해 보시고, 입금 부탁드립니다."` 와 같은 형태로 구성하며, 계좌 정보 뒤의 최하단에 `[참고] 전자계산서 발행일` 목록을 배치합니다.
- **발송 UI 노출 정보 및 유효성**:
  - 발송 모달 내에 대표자명, 담당자 성명, 직위 및 연락처를 한눈에 볼 수 있도록 표시합니다.
  - 각 주기별 상세 미수 내역(예: `2025년 하반기 : 500,000원 | 2026년 상반기 : 450,000원`)을 하단에 병렬로 시각화하여 미수금 합계의 상세 내역을 명확히 제공합니다.
- **연락처 유효성 검사**: 담당자 휴대폰 번호(`phone`)가 존재하지 않거나, 한국 휴대폰 규격(010/011/016/017/018/019로 시작하는 10~11자리 숫자)이 아닐 경우 테이블 내 문자 발송 버튼을 비활성화하고 '담당자 연락처 없음' 안내 툴팁을 표시합니다.
- **무료 문자 발송 및 통장사본 가이드**: 모바일 기기의 무료 문자 요금제를 적극 활용할 수 있도록 `sms:` 링크 방식(텍스트 전용)을 제공하며, 모바일 브라우저의 파일 공유 기능을 활용한 **[📲 이미지와 함께 전송 (공유)]** 버튼을 제공하여 스마트폰 문자 앱에 통장사본 이미지 파일과 본문을 동시에 첨부해 전송할 수 있도록 돕습니다. 보조 수단으로 클립보드 이미지 복사 기능 및 다운로드 기능을 함께 제시합니다.
- **권한 제어**: 로그인한 사용자의 역할이 **관리자**(`user?.role === "관리자"`)일 때만 `안내문자` 버튼이 노출되며, 일반 사용자는 해당 버튼을 볼 수 없도록 차단합니다.
