# 시스템 핵심 비즈니스 로직 가이드 (The Joo Rule)

이 문서는 측정일지 시스템의 데이터 무결성을 유지하고 회귀 오류를 방지하기 위한 핵심 비즈니스 규칙을 정리한 것입니다.

---

## 1. [The Joo Rule] 지청별 예외 발행일 규칙
특정 조건의 사업장에 대해 전자계산서 발행일을 자동으로 계산합니다.

- **대상 영역**: 지정지청이 **'대전지청'** 또는 **'천안지청'**인 경우
- **대상 업종**: 업종분류가 **'공업사'**인 경우
- **규칙**: 전자계산서 발행일(`electronic_invoice_date`)을 **측정일 익일(워킹데이 기준)**로 자동 기록
- **적용 시점**: 2026-04-11 이후 측정 건부터 적용 (소급 적용 금지)
- **워킹데이 산정**: 주말(토, 일) 및 한국 법정 공휴일을 제외한 다음 영업일

## 2. 데이터 동기화 정책 (Latest Wins)
엑셀 데이터와 시스템 데이터 간의 충돌 발생 시 처리 원칙입니다.

- **원칙**: 엑셀 원본(`측정사업장.xls`, `사업장정보.xls`)에 데이터가 존재할 경우, 시스템 내 기존 필드 값을 **무조건 덮어씀**
- **대상 필드**: 사업자번호, 산재번호, 개시번호, 근로자수, 이메일, 담당자 연락처 등
- **목적**: 전산망 데이터의 원본성을 시스템 전체에 유지하여 정보 파편화 방지

## 3. 멀티 데이트(Multi-Date) 측정 관리
불연속적인 다수 일자의 측정 건을 관리하는 규칙입니다.

- **측정일수 계산**: 시작일과 종료일 사이의 기간 중 **주말 및 공휴일을 제외한 실제 작업일(영업일)**만 카운트
- **공시료 코드**: 멀티 데이트 발생 시 각 일자별 공시료 코드를 "4/21: B, 4/22: C" 형식으로 병합 관리
- **측정자**: 여러 명의 측정자가 참여할 경우 콤마(,)로 구분하여 통합 기록

## 4. 연번 부여 알고리즘
- **5인 이상 사업장**: 연번을 1씩 증가시키며 고유 번호 부여
- **5인 미만 사업장**: 직전 사업장에 부여된 연번을 재사용(중복 허용)하여 행정 관리 일관성 유지

## 5. 국고지원 상태값 정규화
- **내부 표준값**: **'대상'**, **'비대상'** (지원, 미지원 등 변종 값 지양)
- **UI 표기**: 항목명은 **'국고지원'**을 유지하여 사용자 익숙함 보호
- **데이터 변환**: 엑셀 업로드 시 '지원', '지원대상' -> '대상'으로, '미지원' -> '비대상'으로 자동 정규화
- **전파 규칙**: `measurement_target_business`의 최신 국고 상태는 일지(`measurement_journal`) 생성 및 보정 시 항상 최우선으로 반영됨

## 6. 일련번호 자동 부여 제외 (기타매출) 처리 규칙
'일련번호 자동 부여 제외'를 선택한 측정 건을 일반 측정비와 분리하여 관리합니다.

- **분류 트리거**: `revenue_type === '기타매출'` (UI의 '일련번호 자동 부여 제외' 체크 시)
- **리스트 노출**: 매출관리 화면의 '측정비' 탭에서 제외하며, **'기타'** 탭의 매출 내역 리스트에 포함함
- **집계 원칙**:
    - 매출 집계 및 년도별 통계 테이블에서 지사별(대전, 천안 등) 측정비가 아닌 **'기타'** 카테고리로 합산함
    - 미수금 상세 내역 조회 시 '기타' 카테고리 내에 해당 일지들이 포함되어야 함
- **수정 로직**: 기타 매출 내역에서 일지 기반 항목(`source: "journal"`) 수정 시, 일반 기타 매출 모달이 아닌 **기존 측정일지 수정 폼**을 호출하여 데이터의 원본성을 유지함
