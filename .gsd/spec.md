# 국고지원금 입금 정보 일괄 업로드(정산) 기능 구현 스펙 (매출 관리 화면)

## 1. 개요
- 국고지원 대상 사업장의 실제 입금 내역(엑셀)을 '매출 상세 현황'의 데이터와 대조하여, **5개 항목이 완벽히 일치하는 경우에만** 입금 정보를 자동으로 등록(정산)한다.
- 데이터 정합성을 최우선으로 하며, 처리 과정을 실시간으로 확인할 수 있는 '신호등' 인디케이터를 제공한다.
- **권한 관리**: 해당 기능은 '일지 담당자' 권한을 가진 사용자에게만 노출 및 허용된다.

## 2. 주요 기능 및 요구사항
1. **업로드 버튼 위치**:
   - 매출 관리 > 매출 상세 현황 > 측정비 화면의 우측 **'필터 초기화' 버튼 바로 좌측**에 배치한다.

2. **데이터 검증 및 입력 로직 (5대 일치 요건)**:
   - 시스템의 측정일지 데이터와 다음 5개 항목을 대조:
     1. **측정년도**
     2. **주기**
     3. **국고지원 여부** (반드시 '대상'이어야 함)
     4. **입금일(국고)**
     5. **입금액(국고)** (가장 중요, 1원이라도 다르면 저장 불가)

3. **실시간 처리 현황 (신호등 표시)**:
   - 업로드 진행 시, 각 행(Row)의 맨 좌측에 상태 표시용 인디케이터를 추가한다.
   - **🟢 초록**: 성공 (데이터 저장 완료)
   - **🔴 빨강**: 실패 (금액 불일치 등 오류 발생)
   - **🟡 노랑**: 진행 중 (현재 API 처리 중)
   - **⚪ 회색**: 대기 (파일 파싱 후 대기 상태)

4. **오류 처리**:
   - 금액 불일치 시 DB 저장 절대 불가.
   - 실패 사유는 해당 행의 신호등에 툴팁 또는 별도 리스트로 사용자에게 상세 피드백.
