# [Spec] 측정 대상 사업장 '실시여부' 상태 동기화 강화

## 1. 개요
예비조사 데이터를 삭제하거나 측정 정보를 제거했을 때, 측정 대상 사업장 목록의 '실시여부' 상태가 자동으로 '미실시'로 돌아가지 않는 문제를 해결한다.

## 2. 현상 및 원인
- **현상**: 예비조사 일정을 삭제하여 실시일이 비었음에도 불구하고, 사업장 목록에서는 여전히 '실시' 상태로 남아 있음.
- **원인**: 
  - 최근 '자동 판단 로직'을 제거하면서 데이터베이스에 저장된 '실시' 값이 그대로 유지됨.
  - 예비조사 삭제/수정 API와 사업장 수정 API에서 실측 데이터가 사라질 때 상태값을 '미실시'로 되돌려주는 동기화 로직이 누락됨.

## 3. 해결 방안 (동기화 로직 보완)

### A. 예비조사 API 보완 (`app/api/survey/[id]/route.ts`)
- **PUT (수정) / DELETE (삭제)**: 
  - 사업장의 마지막 예비조사 데이터가 사라져서 `measurement_date`가 `null`이 되는 경우, `measurement_target_business`의 `is_registered` 상태를 강제로 **'미실시'**로 변경한다.

### B. 사업장 API 보완 (`app/api/businesses/route.ts`)
- **PATCH (수정)**:
  - 실측 정보(실시일, 담당자 등) 업데이트 시, 최종적으로 도출된 `measurement_end_date`가 `null`이면 현재 상태를 체크한다.
  - 현재 상태가 '실시' 또는 '확정'인데 측정 정보가 하나도 없다면 자동으로 **'미실시'**로 변경하여 저장한다.

## 4. 세부 규칙
- **수동 설정 존중**: 사용자가 명시적으로 '거래종료' 등을 선택한 경우는 자동 동기화에서 제외하여 주님의 의사결정을 보호한다.
- **조건부 동기화**: '실시' 상태임에도 '날짜'가 없는 모순된 상황만 자동으로 바로잡는다.

## 5. 승인 요청
