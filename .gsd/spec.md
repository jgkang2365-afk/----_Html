# [국고지원 -> 국고대상] 코드 정규화 및 로직 보강 계획

모든 DB 테이블의 `national_support_status` 값이 '지원'에서 '대상'으로 마이그레이션됨에 따라, 프론트엔드 UI의 문자열을 변경하고 엑셀 동기화 로직 및 API 로직을 보강하여 데이터 일관성을 확보합니다.

## 1. 개요
- **목적**: '지원'이라는 용어를 '대상'으로 통일하여 사용자 혼란을 방지하고 DB 제약 조건과 일치시킴.
- **주요 변경 사항**:
    - 프론트엔드 UI의 '지원' 문자열을 '대상'으로 교체.
    - 엑셀 파싱 시 '지원', '지원대상'을 '대상'으로 자동 변환하는 로직 추가.
    - 백엔드 API에서 불필요한 '지원' 변환 로직 제거 또는 수정.
    - 최신 국고 정보가 측정일지에 누락 없이 반영되는지 검증.

## 2. 상세 작업 계획

### 2.1 프론트엔드 UI 및 백엔드 API 문자열 교체
- **JournalEditForm.tsx**: 
    - "국고지원 여부" -> "국고대상 여부"
    - "국고지원 상태" -> "국고대상 상태"
- **NationalSupportManagement.tsx**:
    - "국고지원 상태" -> "국고대상 여부" (또는 "대상 여부")
    - 필터 및 테이블 헤더의 "지원" -> "대상"
- **MeasurementTargetBusinessManagement.tsx**:
    - "국고지원여부" -> "국고대상여부"
- **Dashboard.tsx / DashboardSample.tsx**:
    - 통계 레이블 "지원" -> "대상"
- **app/api/summary/[id]/route.ts**:
    - "대상"을 "지원"으로 변환하던 레거시 로직 제거 (이제 DB가 "대상"을 직접 수용함)

### 2.2 excel-sync.ts 로직 보강
- `lib/sync/excel-sync.ts` 내 `parseMeasurementBusiness` 및 `parseBusinessInfo` (또는 관련 유틸)에서 `national_support_status` 추출 시 정규화 로직 적용.
- **입력값**: "지원", "지원대상", "대상" -> **결과값**: "대상"
- **입력값**: "비대상" -> **결과값**: "비대상"

### 2.3 검증 및 테스트
- `measurement_target_business`의 `national_support_status`가 `measurement_journal`에 생성/업데이트 시 정확히 반영되는지 테스트.
- 엑셀 업로드 테스트: "지원대상"으로 작성된 엑셀 파일이 DB에 "대상"으로 저장되는지 확인.

## 3. 사용자 검토 사항
> [!IMPORTANT]
> "국고지원"이라는 용어 자체를 모두 "국고대상"으로 바꿀지, 아니면 상태값인 "지원"만 "대상"으로 바꿀지 확인이 필요합니다. 
> 사용자의 요청은 '"지원"이라는 문자열을 "대상"으로 교체'하는 것이므로, "국고지원" -> "국고대상", "지원" -> "대상"으로 일괄 적용할 예정입니다.
