# [Spec] 측정 대상 사업장 '실시여부' 초기화 문제 해결

## 1. 개요
측정 대상 사업장 목록에서 사용자가 '미실시'로 설정하여 저장했음에도 불구하고, 다시 조회하거나 페이지를 새로고침하면 '실시'로 자동 변경되는 현상을 해결한다.

## 2. 현상 및 원인
- **현상**: 목록에서 '미실시' 저장 -> 목록에서 정상 표시 -> '조회' 버튼 혹은 새로고침 시 '실시'로 복구.
- **원인**:
  - `app/api/businesses/route.ts`의 `GET` API에서 `preliminary_survey`(예비조사) 레코드가 존재할 경우 `is_registered` 상태를 강제로 '실시'로 변환하는 로직이 있음.
  - 신규 등록(`POST`) 시 예비조사 레코드가 기본값으로 자동 생성되어 모든 신규 사업장이 '실시'로 오판단되는 구조임.

## 3. 해결 방안
- **조회 로직 수정**: `GET` API에서 데이터베이스에 저장된 `is_registered` 값을 최우선으로 존중하도록 수정. 예비조사 레코드 존재 여부에 따른 강제 전환 로직 제거.
- **정규화 규칙 유지**: `normalizeBusinessStatus`를 통해 기존의 '확정', '미확정' 등 레거시 상태값을 신규 규격('실시', '미실시')으로만 변환하여 표기.

## 4. 세부 사항
- `isRegisteredText === "미실시" && isSurveyRegistered` 조건으로 '실시' 전환하는 부분을 삭제하거나 주석 처리.
- 사용자가 명시적으로 관리하는 상태값이 시스템에 의해 덮어씌워지지 않도록 보장.
