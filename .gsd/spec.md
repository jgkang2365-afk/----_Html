# [Spec] 측정 대상 사업장 데이터 동기화 완전 자동화

## 1. 개요
예비조사 데이터 삭제 및 수정 시, 측정 대상 사업장 목록에 남아 있는 '종료일(~ 날짜)' 및 '측정자 명단' 등 잔여 데이터(Ghost Data)를 완벽하게 제거하고 동기화한다.

## 2. 현상 및 원인
- **현상**: 예비조사 일정을 삭제했음에도 목록의 실시일 칸 아래에 `~ 2026-05-01`과 같은 종료일 정보가 계속 표시됨.
- **원인**: 
  - `app/api/survey/[id]/route.ts`의 삭제/수정 로직에서 `measurement_date`만 업데이트하고, `measurement_end_date` 및 `collaborators` 정보를 갱신하지 않음.
  - 데이터베이스에 stale(오래된) 데이터가 남아 있어 UI에서 이를 렌더링하고 있음.

## 3. 해결 방안 (Full Re-Sync 로직 도입)

### A. 예비조사 API 고도화 (`app/api/survey/[id]/route.ts`)
- **수정/삭제 시 통합 동기화**:
  1. 해당 사업장(`code`)의 현재 주기에 남은 모든 예비조사 데이터를 재조회한다.
  2. 남은 데이터가 **있을 때**:
     - 가장 빠른 날짜 -> `measurement_date` (시작일)
     - 가장 늦은 날짜 -> `measurement_end_date` (종료일)
     - 모든 측정자 합집합 -> `collaborators` (협력자 명단)
     - 위 정보를 `measurement_target_business` 테이블에 동시에 업데이트한다.
  3. 남은 데이터가 **없을 때**:
     - 시작일, 종료일, 협력자 명단을 모두 `null`로 초기화한다.
     - 상태(`is_registered`)를 **'미실시'**로 변경한다.

### B. 사업장 API 보완 (`app/api/businesses/route.ts`)
- **PATCH (수정)**:
  - 정보 제거 시 `measurement_end_date`가 함께 초기화되도록 로직을 재점검한다.

## 4. 기대 효과
- 날짜를 지웠음에도 `~ 2026-05-01`과 같은 찌꺼기 데이터가 보였던 문제 해결.
- 상태와 날짜, 참여 인원 정보가 항상 100% 일치하도록 보장.

## 5. 승인 요청
위 보완된 Full Re-Sync 설계에 대해 승인해 주시면 구현을 진행하겠습니다.
