  2. `is_interactive === False`인 백그라운드 구동 환경에서는 UAC 관리자 권한 상승 시도를 차단하고 에러 메시지와 함께 즉시 `sys.exit(1)`로 조기 종료하도록 조치함.
  3. 글로벌 에러 처리 핸들러에서도 `is_interactive` 조건에 따라 대기 없이 에러 로그만 남긴 뒤 `sys.exit(1)`로 즉시 비정상 종료를 반환하도록 예외 처리를 전면 개정함.

* **해결법:**
  1. 이메일 입력창의 `onBlur` 및 `onChange` 핸들러에서 콤마(`,`) 또는 세미콜론(`;`)과 양끝 공백을 정규식으로 유연하게 스플릿하여 이메일(1)과 이메일(2)로 자동 파싱·할당하고, 이메일 송부 서비스(`email-service.ts`)에서도 이를 수신자(첫 번째 주소)와 참조(두 번째 주소)로 자동 분리하여 발송하도록 비즈니스 로직을 완비함.
  2. 담당자 정보 영역을 **1줄 3열 (성명, 직책, 휴대폰)** 및 **2줄 4열 (이메일1, 2 및 계산서메일1, 2)**의 이중 그리드 구조로 전면 개편하여 가로 공간을 극대화함.

  4. `JournalEditForm.tsx`에서 Input 컴포넌트를 `md:col-span-2` 및 `md:col-span-1`로 선언한 **Wrapper div**로 각각 감싸 3열 그리드가 바깥 div에 확실히 정렬되도록 수정함.
  5. 부모를 **11열 그리드 (`grid-cols-11`)**로 개편하고, 첫째 줄을 **6:3:2 (사업장명 6, 대표자명 3, 총인원 2)** 비율로 재조정하여 총인원의 가로폭을 확보하고, 나머지 줄도 합계 11열(`4 + 4 + 3`, `11`, `5 + 6`)로 완전히 통일하여 모든 행의 우측 끝선 수직 정렬을 칼같이 맞춤. 또한 스핀 화살표를 없애기 위해 총인원 인풋의 type을 text로 바꾸고 숫자 이외 문자를 필터링함.

## 2. 누적된 성공 로직 & 규칙
* **그리드 일치 원칙:** 동일 행에 헬퍼 텍스트 유무가 다른 열이 섞여 있을 때에는 `pb-5` 패딩과 `absolute bottom-0` 위치 지정을 적극 활용해 높이 균형을 유지한다.
* **캘린더 정렬 보정:** 캘린더 입력 필드는 다른 필드와 1:1로 정확하게 일치하도록 전용 클래스나 어긋남을 방지하는 패딩/마진 구조를 공유해야 한다.
* **입력창 높이 통일화:** 개별 인풋에 개별적으로 클래스를 제어하는 대신 `Input` 공통 컴포넌트 자체의 기본 높이를 보정하여 프로젝트 전체 필드 규격을 통일한다.
* **UI 타이틀 문구 간소화:** 사용자 툴팁과 타이틀 속성은 축약되고 단순화된 키워드를 활용해 시인성을 높인다.
* **등록 폼 필드 연동 원칙:** 프론트엔드 모달 등록 폼에 새로운 컬럼을 추가할 때에는 반드시 백엔드 API의 POST/PUT 핸들러가 해당 속성을 명시적으로 파싱하고 DB에 반영하도록 동기화해 주어야 한다.
* **상태 제약 검증:** 상태 필드를 추가/수정할 때는 DB 마이그레이션 및 도메인 제약조건에 선언된 정확한 문자열 값(예: `'대상'`, `'비대상'`)을 사용하여 삽입 오류를 예방한다.
* **마스터 뷰 기반 셀프 데이터 추출 설계:** 복잡한 동적 DB 조인 툴을 브라우저에 직접 드래그앤드롭으로 구축하여 오버엔지니어링하기보다, 백엔드에서 결합된 통합 마스터 데이터셋을 제공하고 사용자는 화면에서 '상세 필터 빌더'와 '출력 컬럼 끄기/켜기' 도구만을 활용해 원하는 형태의 엑셀 보고서를 추출하도록 설계하는 것이 보안과 안정성 면에서 우수하다.
* **공통 UI 컴포넌트 타입 검증:** 커스텀 UI 라이브러리를 사용할 때는 선언된 인터페이스 타입을 사전에 확인하고 정확히 매치하여 컴파일 에러를 차단한다.
* **콤팩트 제어 카드 설계 원칙 (v0.2.6 신설):** 상단 설정 바나 컨트롤러 영역은 패딩을 `p-3` 정도로 작게 조절하고 설명과 제목을 최대한 수평 정렬하여 데이터 테이블이 위치할 세로 공간을 차단한다.
* **제어 요소 가로 배치 보장 (v0.2.6 신설):** 셀렉트박스와 버튼처럼 한 세트로 묶여 있는 제어 요소 그룹은 `flex-nowrap`과 `flex-shrink-0`을 부여하여 해상도가 작아지거나 부모 너비가 제한될 때 줄바꿈이 일어나는 레이아웃 깨짐을 예방한다.

* **크롤러 연동 상태 코드 표준화 (v0.2.8 신설):** 백그라운드 크롤링 프로세스에서 백엔드 API로 결과를 전송할 때는 한글 등 로컬 텍스트 대신 표준화된 영문 상태 코드(예: SUPPORT, NON_SUPPORT, STANDBY)를 사용하여 인코딩 깨짐을 원천 차단한다.
