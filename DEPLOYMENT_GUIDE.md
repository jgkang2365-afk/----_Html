# 배포 서버(Windows 10) 구축 및 운영 가이드

이 문서는 새 컴퓨터(깡통 PC)를 배포 서버로 세팅하고, 프로그램을 안정적으로 운영하기 위한 지침서입니다.

---

## 1. 서버의 의미와 역할
배포 서버는 **24시간 켜져 있는 전용 컴퓨터**입니다. 
- 주님의 개인 PC를 쓰지 않고도 프로그램이 항상 돌아가게 해줍니다.
- 대시보드 접속, K2B 자동 업로드, 이메일 일괄 발송을 대행합니다.

## 2. 서버 구축 전 필수 준비물 (직접 설치 필요)
새 컴퓨터에 다음 프로그램들을 먼저 설치해 주세요.
- [ ] **Google Chrome**: K2B 자동화를 위해 반드시 필요합니다.
- [ ] **Node.js (v20 이상)**: 프로그램 실행 엔진입니다. ([다운로드](https://nodejs.org/))
- [ ] **소스 코드 복사**: 개발된 프로그램 폴더 전체를 서버의 적당한 위치(예: `C:\Work\system`)에 복사합니다.

## 3. 초기 설정 단계 (딱 한 번만 실행)

### 3.1. 자동 설정 스크립트 실행
서버 전용 PowerShell 터미널을 **관리자 권한**으로 열고 아래 명령을 실행합니다.
```powershell
.\scripts\setup-server.ps1
```
> [!TIP]
> 이 스크립트는 크롬 설치 확인, 환경 변수 등록, 네트워크 경로 접근성 체크를 자동으로 도와줍니다.

### 3.2. 환경 변수(.env) 설정
복사된 폴더 내의 `.env` 파일을 메모장 등으로 열어 다음 정보를 입력합니다.
- `REPORT_STORAGE_ROOT`: 보고서가 있는 네트워크 경로 (예: `Z:\data` 또는 `\\Server\data`)
- `NAVER_EMAIL_PW`: 네이버 메일 발송용 앱 비밀번호
- `K2B_ID` / `K2B_PW`: K2B 로그인 정보
- `DATABASE_URL`: Supabase DB 접속 정보

## 4. 프로그램 실행 및 운영

### 4.1. 서버 구동
명령 프롬프트(cmd) 또는 PowerShell에서 다음 명령을 순서대로 입력합니다.
```bash
npm install     # 최초 1회만 실행 (필수 라이브러리 설치)
npm run build   # 실행용 파일 생성
npm run start   # 서버 시작
```

### 4.2. 주요 기능 운영 팁
- **K2B 업로드**: 헤드리스 모드가 적용되어 있어 화면에 창이 뜨지 않아도 작업이 진행됩니다. 진행 상황은 터미널 로그로 확인 가능합니다.
- **네트워크 드라이브**: 만약 `Z:` 드라이브가 끊기면 프로그램이 경고를 띄웁니다. 이 경우 환경 변수 경로를 다시 확인해 주세요.

---

## 5. 문제 해결 (Troubleshooting)
- **메일 발송 실패**: 네이버 SMTP 설정 및 앱 비밀번호를 다시 확인하세요.
- **K2B 업로드 중단**: 서버의 인터넷 연결 상태와 K2B 사이트 점검 여부를 확인하세요.
- **로그 확인**: 프로그램 폴더 내에서 출력되는 실시간 로그(console)를 보시면 상세한 오류 원인을 알 수 있습니다.
