# 환경 변수 설정 안내 (긴급)

## 현재 문제
"Invalid API key" 오류가 발생하고 있습니다. Supabase 환경 변수가 설정되지 않았습니다.

## 빠른 해결 방법

### 1단계: Supabase에서 값 가져오기

1. [Supabase 대시보드](https://supabase.com/dashboard) 접속
2. 프로젝트 선택 (없으면 새로 생성)
3. 왼쪽 메뉴에서 **Settings** (⚙️) 클릭
4. **API** 메뉴 클릭
5. 다음 값들을 복사:
   - **Project URL** (예: `https://xxxxx.supabase.co`)
   - **anon public** 키 (긴 문자열)
   - **service_role** 키 (긴 문자열, 주의: 보안상 중요!)

### 2단계: .env.local 파일 생성

프로젝트 루트 디렉토리에 `.env.local` 파일을 생성하고 다음 내용을 붙여넣으세요:

```env
NEXT_PUBLIC_SUPABASE_URL=여기에_Project_URL_붙여넣기
NEXT_PUBLIC_SUPABASE_ANON_KEY=여기에_anon_public_키_붙여넣기
SUPABASE_URL=여기에_Project_URL_붙여넣기
SUPABASE_SERVICE_ROLE_KEY=여기에_service_role_키_붙여넣기
```

**중요 사항:**
- 따옴표(`"`) 없이 작성
- 등호(`=`) 앞뒤 공백 없음
- 각 줄 끝에 공백 없음

### 3단계: 개발 서버 재시작

1. 터미널에서 개발 서버 중지 (Ctrl+C)
2. 다시 시작:
   ```bash
   npm run dev
   ```

### 4단계: 테스트

브라우저에서 `http://localhost:3000/login` 접속하여 로그인 시도

---

## 임시 해결책: 로그인 없이 동기화 테스트

환경 변수 설정이 어렵다면, 로그인 없이 동기화를 테스트할 수 있습니다:

### 터미널에서 테스트

```bash
# PowerShell에서
Invoke-WebRequest -Uri "http://localhost:3000/api/test-sync" -Method POST

# 또는 curl이 있다면
curl -X POST http://localhost:3000/api/test-sync
```

### 브라우저에서 테스트

1. 주소창에 입력: `http://localhost:3000/api/test-sync`
2. 또는 개발자 도구 콘솔에서:
   ```javascript
   fetch('/api/test-sync', { method: 'POST' })
     .then(res => res.json())
     .then(data => console.log(data));
   ```

## Supabase 프로젝트가 없는 경우

1. [Supabase](https://supabase.com) 접속
2. "Start your project" 또는 "New Project" 클릭
3. GitHub로 로그인 (또는 이메일)
4. 프로젝트 생성:
   - **Name**: 측정일지 관리 시스템
   - **Database Password**: 강력한 비밀번호 설정
   - **Region**: 가장 가까운 지역 선택
5. 프로젝트 생성 완료 대기 (약 2분)
6. 위의 1단계부터 진행

---

## 도움이 필요하신가요?

환경 변수 설정이 완료되면 로그인이 정상적으로 작동합니다.
설정 중 문제가 있으면 알려주세요!
