# 제품 요구사항 정의서 (PRD) - 작업환경측정결과 보고서 처리 통합

## 1. 개요
기존에 개별적으로 운영되던 파이썬 기반의 '작업환경측정결과 보고서 처리' 기능을 현재 웹 기반의 '측정일지 시스템' 내에 통합하여 데이터의 일관성을 확보하고 업무 효율을 극대화한다.

## 2. 목표
- 엑셀 데이터 동기화와 보고서 관리 프로세스의 일원화
- 업체별 보고서 PDF 파일의 자동 탐색 및 이메일 발송 자동화
- 웹 대시보드를 통한 직관적인 처리 현황 모니터링

## 3. 핵심 사용자 스토리
1. **관리자**는 엑셀 동기화 버튼을 통해 최신 업체 정보와 이메일 주소를 DB에 업데이트할 수 있다.
2. **사용자**는 년도와 반기를 선택하여 해당 기간의 측정 업체 목록을 조회할 수 있다.
3. **사용자**는 내비게이션바의 '보고서 처리' 메뉴를 통해 전용 대시보드에 즉시 접근할 수 있다.
4. **사용자**는 년도와 반기를 선택하여 해당 기간의 측정 업체 목록을 조회하고, 특정 조건으로 필터링하거나 업체를 개별/일괄 선택할 수 있다.
5. **사용자**는 특정 업체를 선택하고 '메일 발송' 또는 'K2B 업로드' 버튼을 클릭하여 자동화를 실행하며, 실행 결과(K2B 전송일자 등)는 기존 측정일지 데이터에 실시간으로 자동 반영된다.

## 4. 주요 기능 요구사항
### 4.1 데이터 연동
- `measurement_business.xlsx` 파일에서 이메일 정보(BL열)를 추출하여 DB에 저장
- 업체명 정규화(공백, 괄호 제거 등) 및 규칙 기반 매칭 로직 (`사업장명(보고서-YY상/하).pdf`) 구현

### 4.2 파일 및 메일링 서비스
- Z 드라이브(`Z:\data\...`) 내의 특정 폴더에서 규칙 기반 파일 자동 탐색
- 규칙(`사업장명(보고서-YY상/하).pdf`)에 맞지 않는 보고서 파일은 발송 대상에서 자동 제외
- **수수료 내역서 유연한 첨부**: 해당 업체 폴더 내에 `*수수료 내역서(YY상/하).pdf` 형태의 파일이 존재할 경우 사업장명 일치 여부와 상관없이 자동 첨부
- 네이버 SMTP 고정 아이디(`5678882`) 기반 발송 시스템
- **다중 수신자 지원**: 콤마(,)로 구분된 여러 이메일 주소로 일괄 발송 기능 지원
- **수신자 편집 기능**: 발송 전 대시보드 상에서 수신 이메일 주소를 직접 수정하거나 추가할 수 있는 기능

### 4.4 UI/UX (보고서 처리 대시보드)
- **접근성**: 메인 내비게이션바에 '보고서 처리' 항목을 추가하여 독립된 관리 페이지로 진입
- **필터링 및 선택**: **'보고서 처리' 전용 페이지 상단**에 년도/반기 필터링 및 검색 기능을 배치하고, 리스트 좌측의 체크박스를 통해 대상 업체를 개별/전체 선택
- **결합**: K2B 전송 성공 시 해당 정보(`전송일자`, `전송자`)를 `measurement_journal` 테이블에 자동 업데이트

## 5. 제약 사항 및 고려 사항
- 보안: 네이버 SMTP 사용을 위한 애플리케이션 비밀번호(애플리케이션 비밀번호) 설정 필수
- 환경: 사용자의 로컬 환경에서 실행되므로 Z 드라이브와 인터넷 연결이 보장되어야 함
- 신뢰성: K2B 사이트의 UI 변경 시 자동화 스크립트의 유지보수가 필요할 수 있음
