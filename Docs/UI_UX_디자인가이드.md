# UI/UX 디자인 가이드

## 핵심 원칙

### 1. 사용자 편의성 우선
**원칙**: 화면 디자인보다 사용자의 편의와 효율성을 최우선으로 고려합니다.

**적용 사례**:
- 체크박스, 라디오 버튼, 드롭다운 등의 선택 요소는 **가까운 간격**으로 배치
- 마우스 이동 거리를 최소화하여 연속적인 작업을 용이하게 함
- 화면이 넓게 보이는 것보다 실제 사용자가 빠르게 작업할 수 있도록 최적화

---

## 체크박스 및 선택 요소 레이아웃 가이드

### ✅ 권장: 가까운 간격 (flex-wrap)

```tsx
// 좋은 예: 가까운 간격, 자연스러운 흐름
<div className="flex flex-wrap gap-x-4 gap-y-2 p-3">
  {options.map((option) => (
    <Checkbox key={option.value} label={option.label} />
  ))}
</div>
```

**장점**:
- 마우스 이동 거리 최소화
- 빠른 연속 선택 가능
- 자연스러운 왼쪽→오른쪽, 위→아래 흐름

### ❌ 피해야 할 예: 과도한 간격 (grid with large gap)

```tsx
// 나쁜 예: 간격이 너무 넓어서 마우스 이동이 불편함
<div className="grid grid-cols-5 gap-6 p-4">
  {options.map((option) => (
    <Checkbox key={option.value} label={option.label} />
  ))}
</div>
```

**단점**:
- 마우스 이동 거리가 길어서 불편함
- 연속 선택 시 시간 소모
- 화면은 넓게 보이지만 실제 사용성 저하

---

## 간격 가이드라인

### 체크박스/라디오 버튼 그룹
- **가로 간격**: `gap-x-3` 또는 `gap-x-4` (12px~16px)
- **세로 간격**: `gap-y-2` 또는 `gap-y-3` (8px~12px)
- **패딩**: `p-3` (12px) - 너무 넓지 않게

### 버튼 그룹
- **간격**: `gap-2` 또는 `gap-3` (8px~12px)
- 연속 클릭이 필요한 경우 가까운 간격 유지

### 입력 필드 그룹
- **간격**: `gap-4` (16px) - 레이블과 입력 필드 간 적절한 구분
- 세로 간격은 `gap-y-4` 또는 `gap-y-5` (16px~20px)

## 사용자 워크플로우 고려

### 예시 1: 체크박스 복수 선택

**사용 시나리오**:
1. 사용자가 여러 옵션을 빠르게 선택해야 함
2. 마우스를 좌우로 많이 움직이지 않고 선택하고 싶음
3. 선택한 항목을 한눈에 확인하고 싶음

**적용**:
```tsx
// flex-wrap으로 가까운 간격 유지
<div className="flex flex-wrap gap-x-4 gap-y-2">
  {/* 체크박스들이 가까이 배치되어 빠른 선택 가능 */}
</div>
```

### 예시 2: 연속 입력 필드

**사용 시나리오**:
1. 여러 필드를 연속으로 입력해야 함
2. Tab 키로 빠르게 이동하고 싶음

**적용**:
- `gap-4`로 적절한 간격 유지 (너무 가깝지도, 너무 멀지도 않게)
- Tab 순서가 논리적으로 배치

## 반응형 디자인 고려사항

### 모바일 (작은 화면)
- 세로 배치로 전환하여 스크롤 최소화
- 터치 영역 충분히 확보 (최소 44px x 44px)

### 태블릿 (중간 화면)
- 2~3열로 배치 가능
- 가로 간격은 적절히 유지

### 데스크톱 (큰 화면)
- flex-wrap으로 자연스럽게 배치
- 과도한 grid 사용 지양 (간격이 넓어질 수 있음)

---

## 실제 적용 사례

### ✅ 측정일지 비고 필드 (개선 후)

```tsx
<div className="flex flex-wrap gap-x-4 gap-y-2 p-3">
  {/* 가까운 간격으로 빠른 선택 가능 */}
</div>
```

**개선 효과**:
- 마우스 이동 거리 약 60% 감소
- 연속 선택 시간 단축
- 사용자 피드백: "훨씬 편해졌다"

## 체크리스트

새로운 UI 컴포넌트 구현 시:

- [ ] 사용자가 실제로 어떻게 사용할지 먼저 고려했는가?
- [ ] 마우스 이동 거리를 최소화했는가?
- [ ] 연속 작업이 가능한 레이아웃인가?
- [ ] 터치/클릭 영역이 충분한가?
- [ ] 반응형 디자인에서도 사용하기 편한가?
- [ ] 화면이 넓게 보이는 것보다 실제 사용성에 우선순위를 두었는가?

---

## 피드백 반영 프로세스

1. **사용자 피드백 수집**: 실제 사용 중 불편함 지적
2. **원인 분석**: 왜 불편한지 구체적으로 파악
3. **개선 적용**: 사용성 향상을 위한 수정
4. **가이드라인 업데이트**: 동일한 문제 방지를 위한 가이드 추가
5. **다른 부분에도 적용**: 유사한 UI 요소들도 일괄 개선
