# TASKS (AI 개발 파트너용 프롬프트 설계서)

**프로젝트명**: 측정일지 관리 시스템  
**버전**: v1.0  
**작성일**: 2025-01-27

---

## 개요

이 문서는 AI 코딩 파트너가 즉시 협업을 시작할 수 있도록 구체화되고 실행 가능한 단계별 개발 경로를 제공하는 자연어 형태의 태스크 모음입니다. 각 태스크는 PRD, TRD, User Flow, Database Design, Design System 문서를 참조하여 구현해야 합니다.

---

## M0: 프로젝트 초기화 및 기술 스택 설정

### [M0-T1] [x] 프로젝트 초기화 및 폴더 구조 생성

**컨텍스트 및 목표**:  
Next.js 14+ 기반 프로젝트를 초기화하고, 측정일지 관리 시스템에 적합한 폴더 구조를 생성합니다.

**사용자 스토리**:  
As a 개발자, I want to 프로젝트의 기본 구조를 설정하여, 체계적인 개발을 시작할 수 있습니다.

**기술 명세**:
- Next.js 14+ (App Router 사용)
- TypeScript 설정
- Tailwind CSS 설정
- ESLint, Prettier 설정
- 폴더 구조:
  ```
  /app
    /api          # API Routes
    /dashboard    # 대시보드 페이지
    /survey       # 예비조사 페이지
    /journal      # 측정일지 페이지
    /summary      # 측정정보 요약 페이지
    /sales        # 매출관리 페이지
  /components     # 재사용 가능한 컴포넌트
  /lib            # 유틸리티 함수
  /types          # TypeScript 타입 정의
  /hooks          # 커스텀 훅
  /styles         # 전역 스타일
  ```

**인수 조건**:
- Next.js 프로젝트가 정상적으로 실행됨 (npm run dev)
- TypeScript 컴파일 오류 없음
- Tailwind CSS가 정상적으로 작동함
- ESLint, Prettier 설정 완료

**참조 문서**: TRD 섹션 2 (기술 스택), Design System 섹션 11

**자가 수정 지침**:  
프로젝트 실행 시 오류가 발생하면 package.json의 의존성 버전을 확인하고, Next.js 공식 문서를 참조하여 수정하세요.

### [M0-T3] [x] 데이터베이스 스키마 생성

**컨텍스트 및 목표**:  
Database Design 문서를 기반으로 모든 테이블을 생성하고, 인덱스와 제약조건을 설정합니다.

**사용자 스토리**:  
As a 개발자, I want to 데이터베이스 스키마를 생성하여, 데이터를 구조화하여 저장할 수 있습니다.

**기술 명세**:
- 다음 테이블 생성 (SQL 마이그레이션 파일 작성):
  - business_info (사업장정보)
  - measurement_business (측정사업장)
  - measurement_journal (측정일지) - 가장 중요
  - preliminary_survey (예비조사)
  - measurement_summary (측정정보 요약) - 뷰 또는 테이블
  - sync_log (동기화 로그)
- 외래키 제약조건 설정
- 인덱스 생성 (Database Design 섹션 5 참조)
- 제약조건 설정 (NOT NULL, UNIQUE 등)

**인수 조건**:
- 모든 테이블이 정상적으로 생성됨
- 외래키 제약조건이 올바르게 설정됨
- 인덱스가 생성되고 쿼리 성능이 최적화됨
- 제약조건이 데이터 무결성을 보장함

**참조 문서**: Database Design 전체, 특히 섹션 2 (테이블 상세 설계), 섹션 3 (데이터 무결성 규칙)

**자가 수정 지침**:  
마이그레이션 실행 시 오류가 발생하면 SQL 문법을 확인하고, Supabase SQL Editor에서 직접 실행하여 오류 메시지를 확인하세요.

**사용자 스토리**:

- 로그인 페이지 생성 (app/login/page.tsx)

**기술 명세**:
- 역할 정의: 관리자, 측정팀 직원

## M3: 핵심 기능 개발 (MVP)

### [M3-T1] [x] Excel 파일 동기화 서비스 구현 (FEAT-1)

**컨텍스트 및 목표**:  
사업장정보.xls와 측정사업장.xls 파일을 자동으로 읽어서 데이터베이스에 동기화하는 서비스를 구현합니다.

**사용자 스토리**:  
As a 측정팀 직원, I want to Excel 파일이 업데이트되면 자동으로 시스템에 반영되도록, 최신 데이터를 항상 확인할 수 있습니다.

**기술 명세**:
- Python 스크립트 또는 Node.js 스크립트 생성 (lib/sync/excel-sync.ts):
  - Excel 파일 읽기 (openpyxl 또는 xlsx 라이브러리)
  - 데이터 파싱 및 검증
  - 데이터베이스 업데이트 (UPSERT 로직)
  - 동기화 로그 기록
- 스케줄러 설정 (일일 2회 실행: 오전 9시, 오후 6시)
  - 옵션 1: Vercel Cron Jobs 사용
  - 옵션 2: 별도 서버에서 cron job 실행
- 수동 동기화 API 엔드포인트 생성 (app/api/sync/route.ts)
- 동기화 상태 표시 UI (대시보드에 마지막 동기화 시간 표시)

**인수 조건**:
- Excel 파일이 정상적으로 파싱됨
- 데이터베이스에 올바르게 저장됨
- 동기화 로그가 기록됨
- 스케줄러가 정상적으로 작동함 (또는 수동 동기화가 작동함)
- 동기화 실패 시 오류 메시지가 기록됨

**참조 문서**: TRD 섹션 7 (Excel 동기화 아키텍처), User Flow 섹션 2 (FEAT-1), Database Design 섹션 2.6 (동기화 로그)

**자가 수정 지침**:  
Excel 파싱 오류 발생 시 파일 형식을 확인하고, 열 이름 매핑을 검토하세요. 데이터베이스 업데이트 실패 시 트랜잭션 로직을 확인하세요.
