# 배포 환경 Excel 동기화 최적화 방안

## 현재 상황

- **Excel 파일 위치**: `Z:\data\측정팀\자동화 툴\MES 프로그램 DB`
- **배포 환경**: Vercel (서버리스)
- **문제**: Vercel에서 Z: 드라이브 접근 불가

## 최적화 방안 비교

### 방안 1: 파일 업로드 기능 구현 (권장 ⭐)

**개요**: 사용자가 웹 인터페이스에서 Excel 파일을 업로드하고, Supabase Storage에 저장 후 동기화

**장점**:
- ✅ Vercel 등 클라우드 환경에서도 작동
- ✅ 확장성 좋음 (여러 사용자가 업로드 가능)
- ✅ 파일 버전 관리 가능
- ✅ 네트워크 드라이브 의존성 제거
- ✅ 사용자 친화적 (웹 UI에서 직접 업로드)

**단점**:
- ❌ 개발 필요 (업로드 UI + API)
- ❌ Supabase Storage 비용 발생 가능 (무료 플랜: 1GB)

**구현 방식**:
```
사용자 → 웹 UI → Excel 파일 업로드 → Supabase Storage
                                              ↓
                                    동기화 API에서 파일 다운로드
                                              ↓
                                    메모리에서 파싱 및 DB 동기화
```

**예상 개발 시간**: 2-3일

**비용**: 
- Supabase Storage: 무료 플랜 1GB (충분)
- 초과 시: $0.021/GB/월

---

### 방안 2: 내부 서버 배포 (현실적 대안)

**개요**: Windows 서버에서 Next.js 앱을 실행하고, Z: 드라이브에 직접 접근

**장점**:
- ✅ 현재 코드 그대로 사용 가능 (추가 개발 최소)
- ✅ Z: 드라이브 직접 접근 가능
- ✅ 파일 시스템 직접 접근
- ✅ AutoHotkey 스크립트와 통합 가능

**단점**:
- ❌ 서버 관리 필요 (24/7 실행, 업데이트, 백업)
- ❌ Vercel의 장점 활용 불가 (자동 스케일링, 글로벌 CDN 등)
- ❌ 서버 장애 시 전체 서비스 중단

**구현 방식**:
```
Windows 서버
  ├── Next.js 앱 실행 (PM2 또는 서비스)
  ├── Z: 드라이브 접근
  └── AutoHotkey 스크립트 (선택사항)
```

**예상 개발 시간**: 1일 (서버 설정)

**비용**:
- 서버 하드웨어/클라우드 인스턴스
- Windows Server 라이선스 (필요 시)

---

### 방안 3: 하이브리드 방식

**개요**: 로컬 서버에서 파일을 Supabase Storage에 업로드하고, Vercel 앱에서 Storage의 파일을 동기화

**장점**:
- ✅ Vercel 배포 가능
- ✅ Z: 드라이브 접근 가능 (로컬 서버에서만)
- ✅ 자동화 가능 (스크립트로 파일 업로드)

**단점**:
- ❌ 두 시스템 관리 필요
- ❌ 로컬 서버가 필요
- ❌ 복잡도 증가

**구현 방식**:
```
로컬 서버 (Windows)
  ├── Excel 파일 읽기 (Z: 드라이브)
  ├── Supabase Storage에 업로드
  └── (스크립트로 자동화)
              ↓
Vercel 앱
  ├── Supabase Storage에서 파일 다운로드
  └── 동기화 실행
```

**예상 개발 시간**: 3-4일

### 방안 4: Git LFS (비권장)

**개요**: Excel 파일을 Git LFS에 저장하고, 배포 시 함께 배포

**장점**:
- ✅ 간단한 구현

**단점**:
- ❌ Git 저장소 크기 증가
- ❌ 수동 커밋/푸시 필요
- ❌ 자동화 어려움
- ❌ Vercel 빌드 시간 증가

**권장하지 않음**: Excel 파일은 Git에 저장하지 않는 것이 좋습니다.

## 최종 권장: 방안 1 (파일 업로드 기능)

### 이유

1. **장기적 관점**: 확장성과 유지보수성이 좋음
2. **클라우드 네이티브**: Vercel의 장점을 최대한 활용
3. **사용자 경험**: 웹 UI에서 직접 파일 업로드 가능
4. **비용 효율**: Supabase Storage 무료 플랜으로 충분

### 구현 계획

#### 1단계: Supabase Storage 설정
- Storage 버킷 생성 (`excel-files`)
- 업로드 권한 설정 (인증된 사용자만)

#### 2단계: 파일 업로드 API 구현
- `POST /api/upload/excel` 엔드포인트
- Supabase Storage에 파일 저장
- 파일 메타데이터 저장 (파일명, 업로드 시간 등)

#### 3단계: 동기화 로직 수정
- Storage에서 파일 다운로드 (메모리로)
- 기존 파싱 로직 재사용
- 동기화 실행

#### 4단계: UI 구현
- 파일 업로드 컴포넌트
- 업로드 진행 상태 표시
- 최근 업로드 파일 목록

#### 5단계: 자동화 (선택사항)
- 업로드 후 자동 동기화 옵션
- 스케줄 동기화 (업로드된 파일 기준)

## 대안: 방안 2 (내부 서버 배포)

### 빠른 해결이 필요하다면

- **단기 해결책**: 방안 2 (내부 서버 배포)
- **장기 해결책**: 방안 1 (파일 업로드)로 마이그레이션

### 구현 계획

1. **Windows 서버 준비**
   - Node.js 설치
   - PM2 또는 Windows Service 설정

2. **Next.js 앱 배포**
   - 빌드: `npm run build`
   - 실행: `npm start` 또는 PM2

3. **역방향 프록시 설정** (선택사항)
   - IIS 또는 Nginx
   - 도메인 연결

4. **AutoHotkey 스크립트 연동** (선택사항)
   - 파일 변경 감지
   - 자동 동기화 트리거
