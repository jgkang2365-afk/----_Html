## 1. 문제 정의

### 1.1 현재 상황
측정팀은 Excel 기반의 수작업으로 측정일지를 관리하고 있습니다. 이 과정에서 다음과 같은 문제점들이 발생하고 있습니다:

- **Excel 수작업의 번거로움**: 매번 Excel 파일을 열어서 수정하고 저장하는 반복 작업
- **데이터 충돌**: 약 10명의 사용자가 동시에 작업할 때 파일 충돌 및 데이터 손실 위험
- **복잡한 비즈니스 로직의 수동 처리**: 공문연번, 연번, 5인 이상 연번 등 복잡한 번호 부여 규칙을 수동으로 계산해야 함
- **데이터 동기화 문제**: Excel 파일이 일일 2회 업데이트되지만, 최신 정보를 실시간으로 확인하기 어려움
- **여러 파일 관리의 복잡성**: 사업장정보.xls, 측정사업장.xls 등 여러 파일을 왔다 갔다 하며 작업해야 함

### 1.2 문제의 영향
- 업무 시간 낭비 및 비효율성
- 데이터 오류 발생 가능성 증가
- 팀원 간 정보 공유의 어려움
- 실수로 인한 데이터 덮어쓰기 위험

---

## 2. 목표

### 2.1 비즈니스 목표
- **업무 시간 단축 및 효율성 향상**: Excel 수작업 시간을 최소화하고 자동화를 통해 업무 효율성 극대화
- **데이터 정확성 향상**: 복잡한 비즈니스 로직을 자동화하여 데이터 오류를 거의 발생하지 않도록 함
- **실시간 데이터 동기화**: Excel 파일 업데이트를 자동으로 감지하고 최신 정보를 실시간으로 반영

### 2.2 사용자 목표
- 한 화면에서 모든 정보를 확인하고 작업할 수 있음
- 복잡한 번호 부여 규칙을 신경 쓰지 않고 자동으로 처리됨
- 여러 명이 동시에 작업해도 데이터 충돌이 발생하지 않음
- 간단하고 직관적인 인터페이스로 쉽게 사용 가능

## 3. 사용자 페르소나

### 3.1 주요 사용자
**측정팀 직원 (약 10명)**

- **역할**: 측정일지 작성, 수정, 관리 담당
- **기술 수준**: Excel 사용 경험 있음, 웹 기반 시스템 사용 가능
- **사용 환경**: 주로 사무실 PC에서 작업, 가끔 모바일로 조회
- **주요 업무**:
  - 측정일지 검색 및 수정
  - 예비조사 데이터 입력
  - 대시보드를 통한 현황 파악
  - 매출 현황 확인

### 3.2 사용 시나리오
- **시나리오 1**: 사무실 PC에서 측정일지 작성 및 수정
- **시나리오 2**: 현장에서 모바일/태블릿으로 간단한 조회
- **시나리오 3**: 사무실 PC에서 대시보드로 현황 파악 및 보고서 작성

## 4. 사용자 스토리

### FEAT-1: Excel 자동 동기화
**As a** 측정팀 직원  
**I want to** Excel 파일이 업데이트되면 자동으로 시스템에 반영되도록  
**So that** 최신 데이터를 항상 확인할 수 있고 수동으로 파일을 업로드할 필요가 없음

**인수 조건**:
- 사업장정보.xls, 측정사업장.xls 파일이 일일 2회 업데이트될 때 자동으로 감지
- 업데이트된 데이터가 데이터베이스에 자동 반영
- 동기화 상태를 사용자가 확인할 수 있음

### FEAT-2: 측정일지 검색 및 수정
**As a** 측정팀 직원  
**I want to** 측정년도, 측정주기, 사업장명 등으로 측정일지를 검색하고 수정할 수 있도록  
**So that** 필요한 측정일지를 빠르게 찾아서 업데이트할 수 있음

**인수 조건**:
- 측정년도, 측정주기, 사업장명, 지정한계_관할지청, 주소로 검색 가능
- 직전(가장 최신) 자료를 자동으로 불러옴
- 사업장명에 "*번외*"가 포함된 자료는 검색 결과에서 제외
- 검색 → 선택 → 수정 → 저장 프로세스 지원
- 저장 시 측정년도/측정주기 변경 검증 및 경고

### FEAT-3: 공문연번/연번 자동 부여
**As a** 측정팀 직원  
**I want to** 복잡한 번호 부여 규칙을 자동으로 처리하도록  
**So that** 수동 계산 실수를 방지하고 일관된 번호 체계를 유지할 수 있음

**인수 조건**:
- 지정한계_관할지청별로 공문연번 자동 부여 (천-001, 대-001, 평-001, 경-001)
- 관할지청별 상반기/하반기 구분하여 연번 순차 부여
- 총인원 기준 5인 이상 연번 특수 로직 적용 (5인 미만은 직전 번호 재사용)
- 번호는 자동 생성되며 수동 수정 불가

## 5. 성공 지표

### 5.1 노스스타 지표 (결과 지표)
- **데이터 오류 발생률**: 거의 발생하지 않음 (목표: 월 1건 이하)
- **Excel 수작업 시간 감소율**: 50% 이상 감소
- **사용자 만족도**: 측정팀 직원 10명 중 8명 이상 만족

### 5.2 입력 지표 (행동 지표)
- **일일 활성 사용자 수**: 측정팀 직원 10명 중 평균 8명 이상 사용
- **측정일지 생성/수정 건수**: 주당 평균 측정일지 처리 건수 유지 또는 증가
- **시스템 응답 시간**: 검색 및 저장 작업 3초 이내 완료
- **데이터 동기화 성공률**: 99% 이상

## 7. 가정과 리스크

### 7.1 가정 (Assumptions)
1. **기술 가정**:
   - 사용자들이 웹 브라우저를 사용할 수 있음
   - 인터넷 연결이 안정적으로 유지됨
   - Excel 파일 형식이 변경되지 않음

2. **사용자 가정**:
   - 측정팀 직원들이 기본적인 웹 인터페이스 사용에 익숙함
   - Excel 기반 작업 흐름을 이해하고 있음

3. **데이터 가정**:
   - Excel 파일이 정기적으로 업데이트됨 (일일 2회)
   - 데이터 형식이 일관성 있게 유지됨

### 7.2 리스크 및 완화 전략

| 리스크 | 영향도 | 완화 전략 |
|--------|--------|-----------|
| Excel 파일 형식 변경 | 높음 | 파일 형식 변경 감지 및 알림 기능, 유연한 파싱 로직 구현 |
| 데이터 동기화 실패 | 높음 | 자동 재시도 메커니즘, 수동 동기화 버튼 제공, 동기화 로그 기록 |
| 동시 접속 시 성능 저하 | 중간 | 데이터베이스 최적화, 캐싱 전략, 트랜잭션 관리 |
| 사용자 학습 곡선 | 낮음 | 직관적인 UI 설계, 간단한 사용 가이드 제공 |
| 비용 발생 (호스팅/서비스) | 중간 | 무료 티어 활용, 오픈소스 기술 스택 사용, 비용 모니터링 |
