# H0432 데이터 누락 문제 해결 계획

## 현재 상황

- ✅ **로컬 환경**: H0432 데이터가 정상적으로 동기화 및 검색됨
- ❌ **배포 환경 (Vercel)**: H0432 데이터가 검색 결과에 나타나지 않음
- ✅ **Excel 파일**: 로컬 파일에 H0432 데이터가 확실히 존재함 (사용자 확인)
- ❓ **배포 환경 Excel 파일**: Supabase Storage에 업로드된 파일이 로컬 파일과 다를 가능성

## 가능한 원인 분석

1. **Excel 파일 버전 차이** (가장 가능성 높음)
   - 배포 환경에 업로드된 파일이 오래된 버전일 수 있음
   - 파일 저장 시 문제가 발생했을 수 있음

2. **데이터베이스 데이터 불일치**
   - 기존 데이터와 충돌
   - UPSERT 로직 문제
   - 필드 매핑 오류

3. **환경 차이**
   - Node.js 버전 차이
   - 라이브러리 버전 차이
   - Excel 파싱 로직의 환경별 동작 차이

---

## 해결 계획: 두 가지 옵션

### 옵션 1: 로컬에서 먼저 테스트 (✅ 권장)

**장점:**
- ⚡ 빠르고 안전함
- 🛡️ 데이터 손실 위험 없음
- 🔍 환경 차이를 명확히 확인 가능
- 📊 상세한 로그 확인 가능
- 🔄 실패해도 배포 환경에 영향 없음

**단점:**
- 로컬 환경 설정 필요
- Supabase Storage와의 연동 테스트는 배포 환경 필요

**실행 단계:**

#### 1단계: 로컬 환경 확인

```bash
# 1. 개발 서버 실행
npm run dev

# 2. 브라우저에서 접속
# http://localhost:3000
```

#### 2단계: 로컬에서 Excel 파일 테스트

**방법 A: 로컬 파일 시스템 사용 (빠른 테스트)**

현재 코드는 Supabase Storage에서 파일을 읽도록 되어 있지만, 로컬 테스트를 위해:

1. **로컬 Excel 파일 확인**
   - 프로젝트 루트에 `측정사업장.xlsx` 파일 배치
   - H0432가 포함된 최신 버전인지 확인

2. **로컬 동기화 테스트**
   - 대시보드에서 수동 동기화 실행
   - 또는 API 직접 호출: `POST http://localhost:3000/api/sync?type=measurement-business`

3. **결과 확인**
   - 터미널 로그 확인 (상세한 디버그 정보)
   - H0432 검색 테스트: `GET http://localhost:3000/api/journal/search?code=H0432`

**방법 B: Supabase Storage 파일 사용 (배포 환경과 동일)**

1. **최신 Excel 파일 업로드**
   - 대시보드 → 파일 업로드
   - H0432가 포함된 최신 `측정사업장.xlsx` 파일 업로드

2. **로컬에서 동기화**
   - 대시보드에서 수동 동기화 실행
   - 로컬 터미널에서 상세 로그 확인

3. **결과 확인**
   - 로컬 검색 API 테스트
   - H0432 데이터 확인

#### 3단계: 문제 진단

**로컬에서 성공한 경우:**
- 환경 차이로 인한 문제임을 확인
- 배포 환경의 Excel 파일을 최신 버전으로 교체
- 배포 환경에서 다시 동기화

**로컬에서도 실패한 경우:**
- 코드 문제임을 확인
- 로컬 로그를 기반으로 코드 수정
- 수정 후 로컬에서 재테스트

#### 4단계: 배포 환경 적용

로컬에서 성공한 경우:

1. **최신 Excel 파일 업로드**
   - 배포 사이트 대시보드에서 파일 업로드
   - H0432가 포함된 최신 파일 확인

2. **배포 환경에서 동기화**
   - 배포 사이트 대시보드에서 수동 동기화 실행
   - Vercel 로그 확인

3. **검색 테스트**
   - 배포 사이트에서 H0432 검색
   - 결과 확인

### 옵션 2: 데이터베이스 초기화 후 재동기화 (⚠️ 주의 필요)

**장점:**
- 🔄 깨끗한 상태에서 시작
- 기존 데이터 충돌 문제 해결
- 확실한 재동기화

**단점:**
- ⚠️ **모든 데이터 손실** (사업장정보, 측정사업장, 측정일지 등)
- 🔒 **복구 불가능** (백업이 없다면)
- ⏱️ 모든 데이터 재입력 필요
- 📝 사용자 데이터도 삭제됨 (users 테이블 제외)

**실행 단계:**

#### ⚠️ 주의사항

**데이터 백업 필수!**

데이터를 삭제하기 전에 반드시 백업하세요:

1. **Supabase 대시보드에서 데이터 내보내기**
   - Table Editor → 각 테이블 → Export
   - 또는 SQL Editor에서 데이터 백업

2. **중요 테이블 목록:**
   - `business_info` (사업장정보)
   - `measurement_business` (측정사업장)
   - `measurement_journal` (측정일지)
   - `preliminary_survey` (예비조사)
   - `measurement_summary` (측정정보 요약)
   - `users` (사용자 - 삭제하지 않음!)

#### 1단계: 데이터 삭제 (Supabase SQL Editor 사용)

1. **Supabase 대시보드 접속**
   - https://supabase.com/dashboard
   - 프로젝트 선택

2. **SQL Editor 열기**
   - 왼쪽 메뉴 → SQL Editor
   - New query 클릭

3. **다음 SQL 실행** (⚠️ 주의: 모든 데이터가 삭제됩니다!)

```sql
-- ⚠️ 경고: 이 쿼리는 모든 데이터를 삭제합니다!
-- 실행 전에 반드시 백업하세요!

-- 외래 키 제약조건 때문에 순서가 중요합니다
DELETE FROM measurement_summary;
DELETE FROM preliminary_survey;
DELETE FROM measurement_journal;
DELETE FROM measurement_business;
DELETE FROM business_info;
-- sync_log는 로그이므로 선택적으로 삭제
-- DELETE FROM sync_log;
-- users는 삭제하지 않음!
```

4. **확인**
   - Table Editor에서 각 테이블이 비어있는지 확인

#### 2단계: 최신 Excel 파일 업로드

1. **대시보드에서 파일 업로드**
   - H0432가 포함된 최신 `측정사업장.xlsx` 파일 업로드
   - `사업장정보.xlsx` 파일도 업로드 (필요한 경우)

2. **파일 확인**
   - 업로드된 파일명과 날짜 확인

#### 3단계: 데이터 동기화

1. **대시보드에서 수동 동기화**
   - "수동 동기화" 버튼 클릭
   - 또는 API 호출: `POST /api/sync`

2. **동기화 로그 확인**
   - Vercel 로그 확인
   - 처리된 레코드 수 확인

#### 4단계: H0432 검색 테스트

1. **검색 API 테스트**
   - `GET /api/journal/search?code=H0432`
   - 또는 대시보드에서 검색

2. **결과 확인**
   - H0432 데이터가 나타나는지 확인

## 추가 참고사항

### 로컬 환경 설정이 안 되어 있는 경우

`.env.local` 파일이 없는 경우:

1. **Supabase 프로젝트 정보 확인**
   - Supabase 대시보드 → Settings → API
   - Project URL, anon key, service_role key 복사

2. **`.env.local` 파일 생성**

```env
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
SUPABASE_URL=https://xxxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

3. **개발 서버 재시작**

```bash
# 서버 중지 (Ctrl+C)
npm run dev
```

### 로컬에서 Supabase Storage 파일 사용

현재 코드는 Supabase Storage에서 파일을 읽도록 되어 있습니다.
로컬에서 테스트할 때도 배포 환경과 동일하게 테스트하려면:

1. 대시보드에서 파일 업로드 (Supabase Storage에 저장)
2. 로컬에서 동기화 API 호출
3. 로컬 터미널에서 상세 로그 확인
