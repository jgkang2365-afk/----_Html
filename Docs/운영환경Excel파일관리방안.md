# 운영 환경 Excel 파일 관리 방안

## 현재 상황

- **원본 파일 위치**: `Z:\data\측정팀\자동화 툴\MES 프로그램 DB`
- **파일명**: `사업장정보.xls`, `측정사업장.xls`
- **업데이트 주기**: 일일 2회 (자동 업데이트)
- **현재 코드**: 프로젝트 루트의 Excel 파일을 읽도록 설계

## 질문: Z:\ 경로의 파일도 .xlsx로 변환해야 하나요?

### 답변: **운영 환경 구성에 따라 다릅니다**

## 운영 환경별 권장 사항

### 시나리오 1: 로컬/내부 서버에서 실행 (권장)

**환경**: Windows 서버에서 Next.js 앱을 직접 실행하거나, 내부 네트워크에서만 접근

**방안**:
1. **Z:\ 경로의 파일은 그대로 두고 (.xls)**, 코드가 두 형식 모두 지원하도록 설정 ✅
2. 또는 Z:\ 경로의 파일도 .xlsx로 변환 (MES 프로그램이 .xlsx를 생성하도록 변경)

**장점**:
- 코드가 이미 .xls와 .xlsx 모두 지원하므로 추가 작업 불필요
- MES 프로그램이 .xls를 생성한다면 그대로 사용 가능

**단점**:
- .xls 파일은 xlsx 라이브러리에서 읽기 어려울 수 있음 (현재 문제)

### 시나리오 2: Vercel 등 클라우드 배포

**환경**: Vercel, AWS 등 클라우드 환경에 배포

**문제**: Z:\ 경로는 로컬 네트워크 드라이브이므로 클라우드에서 접근 불가 ❌

**해결 방안**:

#### 방안 A: 파일 복사 스크립트 (로컬 서버 필요)
```typescript
// Windows 스케줄러 또는 서비스로 실행
// Z:\ 경로에서 프로젝트 폴더로 파일 복사
// .xls → .xlsx 변환 또는 그대로 복사
```

#### 방안 B: 파일 업로드 기능 구현 (권장)
- 웹 인터페이스에서 Excel 파일 업로드
- Supabase Storage 또는 S3에 저장
- API에서 클라우드 저장소의 파일 읽기

#### 방안 C: 내부 서버에서 실행
- Windows 서버에 Next.js 앱 배포
- Z:\ 경로 직접 접근 가능
- 파일 시스템 기반 동기화

## 권장 사항

### 단기 (개발/테스트 단계)

1. **로컬 프로젝트 폴더의 파일은 .xlsx로 변환 완료** ✅
2. **Z:\ 경로의 파일은 일단 그대로 유지**
3. 코드가 두 형식 모두 지원하도록 구현 (완료) ✅

### 중기 (운영 환경 구축)

**운영 환경이 로컬/내부 서버인 경우**:
- Z:\ 경로의 파일을 .xlsx로 변환하는 것을 권장
- 또는 MES 프로그램이 .xlsx 형식으로 생성하도록 설정 요청

**운영 환경이 클라우드인 경우**:
- 파일 업로드 기능 구현
- 또는 파일 복사 스크립트 + 내부 서버 사용

## Z:\ 경로 파일 변환 시 고려사항

### 변환해야 하는 경우

1. **MES 프로그램이 .xlsx를 생성하도록 변경 가능한 경우**
   - 가장 깔끔한 해결책
   - 일관된 파일 형식 유지

2. **수동 변환이 가능한 경우**
   - 파일이 자동 업데이트되므로, 변환 후에도 계속 업데이트될 것
   - 자동 변환 스크립트가 필요할 수 있음

### 변환하지 않아도 되는 경우

1. **코드가 .xls를 지원하는 경우**
   - 현재 코드는 두 형식 모두 지원
   - 하지만 .xls 파일 읽기에 문제가 있을 수 있음

2. **파일 복사/변환 프로세스를 구축하는 경우**
   - Z:\ → 프로젝트 폴더로 자동 복사
   - 복사 시 자동 변환

## 최종 답변

**현재 단계에서는 Z:\ 경로의 파일 변환이 필수는 아닙니다.**

**이유**:
1. 코드가 이미 두 형식 모두 지원하도록 구현됨
2. 운영 환경 구성이 아직 결정되지 않음
3. 로컬 프로젝트 폴더의 파일만으로도 개발/테스트 가능

**운영 환경이 결정된 후**:
- 로컬/내부 서버: Z:\ 파일도 .xlsx로 변환 권장
- 클라우드: 파일 업로드 기능 또는 복사 스크립트 필요
