# 비고 체크박스 기능 테스트 가이드

## 테스트 환경 선택

### ✅ 권장: 로컬 환경에서 먼저 테스트

**이유:**
1. **빠른 피드백**: 코드 변경 후 즉시 확인 가능
2. **안전한 테스트**: 배포 환경에 영향을 주지 않음
3. **디버깅 용이**: 브라우저 개발자 도구로 쉽게 확인
4. **반복 테스트**: 여러 번 수정하고 테스트하기 쉬움

### 배포 환경 테스트는 언제?

로컬에서 다음을 확인한 후:
- ✅ 체크박스가 정상적으로 표시됨
- ✅ 복수 선택이 정상 작동함
- ✅ 저장 후 다시 불러올 때 선택 상태가 유지됨
- ✅ 기존 데이터(단일 값)도 정상 표시됨

## 로컬 테스트 절차

### 1단계: 개발 서버 실행

```bash
# 프로젝트 루트 디렉토리에서
npm run dev
```

**예상 출력:**
```
▲ Next.js 14.2.0
- Local:        http://localhost:3000
- Ready in 2.5s
```

### 2단계: 브라우저에서 접속

1. **로그인**
   - http://localhost:3000
   - 로그인 페이지에서 로그인

2. **측정일지 페이지 접속**
   - 측정일지 탭 클릭
   - 또는 직접: http://localhost:3000/journal

### 3단계: 측정일지 검색 및 선택

1. **검색 조건 입력**
   - 측정년도, 측정주기 등으로 검색
   - 검색 결과에서 측정일지 선택

2. **편집 모달 열기**
   - "선택" 버튼 클릭
   - 측정일지 편집 폼이 열림

### 4단계: 비고 체크박스 확인

**확인 사항:**

1. **UI 표시 확인**
   - ✅ "비고 (복수 선택 가능)" 레이블이 표시됨
   - ✅ 5개의 체크박스가 표시됨:
     - 최초실시
     - 공정 수시변경
     - 소음 85 이상
     - 전회 미실시
     - 타기관 신규
   - ✅ 체크박스가 그리드 레이아웃으로 정렬됨

2. **기존 데이터 표시 확인**
   - 기존 측정일지의 비고 값이 있는 경우:
     - 단일 값: 해당 체크박스만 체크됨
     - 콤마로 구분된 값: 여러 체크박스가 체크됨
   - 비고 값이 없는 경우: 모든 체크박스가 해제됨

3. **복수 선택 테스트**
   - 여러 체크박스를 선택
   - 선택/해제가 정상 작동하는지 확인
   - 선택된 항목들이 유지되는지 확인

4. **저장 테스트**
   - 여러 항목 선택 후 "수정" 또는 "등록" 버튼 클릭
   - 저장 성공 메시지 확인
   - 저장 후 다시 불러올 때 선택 상태가 유지되는지 확인

### 5단계: 브라우저 개발자 도구 확인

**F12** 키를 눌러 개발자 도구 열기:

1. **Console 탭**
   - 에러 메시지가 없는지 확인
   - 저장 시 네트워크 요청 확인

2. **Network 탭**
   - 저장 요청(`PUT /api/journal/[id]` 또는 `POST /api/journal`) 확인
   - 요청 본문에서 `note` 필드가 콤마로 구분된 문자열인지 확인
     - 예: `"note": "최초실시,소음 85 이상,전회 미실시"`

### 6단계: 데이터베이스 확인 (선택사항)

Supabase 대시보드에서 확인:

1. **measurement_journal 테이블 조회**
2. **note 필드 확인**
   - 저장된 값이 콤마로 구분된 문자열인지 확인
   - 예: `최초실시,소음 85 이상`

## 테스트 체크리스트

### 기본 기능
- [ ] 체크박스 5개가 정상적으로 표시됨
- [ ] 각 체크박스의 레이블이 정확함
- [ ] 체크박스를 클릭하면 선택/해제됨
- [ ] 여러 체크박스를 동시에 선택 가능함

### 데이터 처리
- [ ] 기존 단일 값이 정상적으로 표시됨
- [ ] 기존 콤마로 구분된 값이 정상적으로 표시됨
- [ ] 선택한 항목들이 저장됨
- [ ] 저장 후 다시 불러올 때 선택 상태가 유지됨
- [ ] 아무것도 선택하지 않으면 null로 저장됨

### UI/UX
- [ ] 레이아웃이 깔끔하게 표시됨
- [ ] 반응형 디자인이 정상 작동함 (모바일/태블릿/데스크톱)
- [ ] 체크박스 클릭 영역이 충분함

### 에러 처리
- [ ] 브라우저 콘솔에 에러가 없음
- [ ] 저장 시 에러가 발생하지 않음
- [ ] 잘못된 데이터 형식도 안전하게 처리됨

## 예상되는 문제 및 해결 방법

### 문제 1: 체크박스가 표시되지 않음

**원인**: Checkbox 컴포넌트 import 누락 또는 경로 오류

**해결**:
```typescript
// components/features/JournalEditForm.tsx 상단 확인
import { Checkbox } from "@/components/ui/Checkbox";
```

### 문제 2: 선택한 항목이 저장되지 않음

**원인**: 데이터 변환 로직 오류

**확인 사항**:
- 브라우저 개발자 도구 Network 탭에서 요청 본문 확인
- `note` 필드가 배열이 아닌 문자열로 전송되는지 확인

### 문제 3: 기존 데이터가 표시되지 않음

**원인**: 파싱 로직 오류

**확인 사항**:
- 브라우저 개발자 도구 Console에서 `formData.note` 값 확인
- 콤마로 구분된 문자열이 배열로 변환되는지 확인
