# 404 에러 해결 가이드

## 404 에러란?

404 에러는 "Not Found"를 의미하며, 요청한 리소스(페이지, API 등)를 서버에서 찾을 수 없을 때 발생합니다.

## 404 에러 확인 방법

### 1. 브라우저 개발자 도구 확인

1. **F12** 또는 **우클릭 → 검사**로 개발자 도구 열기
2. **Network (네트워크)** 탭 클릭
3. 페이지 새로고침 또는 작업 수행
4. 빨간색으로 표시된 요청 확인
5. 해당 요청을 클릭하여 다음 정보 확인:
   - **Request URL**: 어떤 URL이 404를 반환하는지
   - **Status Code**: 404
   - **Response**: 에러 메시지

### 2. Console (콘솔) 탭 확인

콘솔 탭에서 에러 메시지를 확인할 수 있습니다.

## 일반적인 404 에러 원인

### 1. API 라우트가 존재하지 않는 경우

**증상**: `/api/upload/excel` 등의 API 엔드포인트에서 404 발생

**원인**:
- 파일이 배포되지 않음
- 빌드 실패
- 파일 경로 오류

**해결 방법**:
1. 배포 환경인 경우: Vercel 등에서 빌드 로그 확인
2. 로컬 환경인 경우: 개발 서버 재시작
3. 파일이 실제로 존재하는지 확인

### 2. Next.js 라우트 구조 문제

**증상**: 페이지나 API가 예상과 다르게 동작

**원인**:
- 파일 위치가 잘못됨
- 파일명 오류

**해결 방법**:
- Next.js App Router 구조 확인
- 파일이 올바른 위치에 있는지 확인

### 3. 빌드 실패

**증상**: 배포 후 여러 기능이 동작하지 않음

**원인**:
- TypeScript 오류
- 빌드 에러

**해결 방법**:
1. 로컬에서 빌드 테스트:
   ```bash
   npm run build
   ```
2. 빌드 로그에서 오류 확인
3. 오류 수정 후 재배포

## 배포 환경에서 404 해결 방법

### Vercel 배포 환경

1. **Vercel 대시보드 확인**
   - 프로젝트 → **Deployments** 탭
   - 최신 배포의 상태 확인
   - 빌드 로그 확인 (오류 메시지)

2. **빌드 로그에서 오류 확인**
   - TypeScript 오류
   - 모듈 import 오류
   - 파일 경로 오류

3. **재배포**
   - 코드 수정 후 Git push
   - Vercel이 자동으로 재배포

## 로컬 환경에서 404 해결 방법

1. **개발 서버 재시작**
   ```bash
   # 서버 중지 (Ctrl+C)
   npm run dev
   ```

2. **빌드 캐시 삭제**
   ```bash
   rm -rf .next
   npm run dev
   ```

3. **파일 확인**
   - API 파일이 `app/api/` 디렉토리에 있는지 확인
   - 파일명이 `route.ts`인지 확인

## 자주 발생하는 404 에러

### `/api/upload/excel` 404 에러

**확인 사항**:
1. `app/api/upload/excel/route.ts` 파일이 존재하는지
2. 파일이 Git에 커밋되어 있는지 (배포 환경인 경우)
3. 빌드 로그에 오류가 없는지

**해결 방법**:
1. 파일 존재 확인
2. 개발 서버 재시작 (로컬)
3. 재배포 (배포 환경)

## 디버깅 팁

### 1. Network 탭에서 확인

```
요청 URL: https://your-site.vercel.app/api/upload/excel
상태 코드: 404
메서드: POST
```

### 2. Console 탭에서 확인

```
Failed to load resource: the server responded with a status of 404 ()
/api/upload/excel:1
```

### 3. 서버 로그 확인

- 로컬: 터미널 출력
- 배포: Vercel Functions 로그
