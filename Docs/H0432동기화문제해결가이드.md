# H0432 데이터 동기화 문제 해결 가이드

## 문제 상황

- `business_info` 테이블에는 H0432 데이터가 있음 (1건)
- `measurement_business` 테이블에는 H0432 데이터가 없음 (0건)
- Excel 파일의 필수 필드는 모두 정상

## 확인 사항

### 1. 수동 동기화 실행 여부 확인

대시보드에서 **"수동 동기화"** 버튼을 클릭했는지 확인하세요.

### 2. 동기화 로그 확인

동기화 후 아래 URL로 동기화 로그를 확인하세요:

```
https://html-tan-six.vercel.app/api/sync
```

(브라우저 주소창에 직접 입력)

**확인할 내용:**
- `results` 배열에서 `measurement_business` 항목 확인
- `records_processed`: 처리된 레코드 수
- `records_inserted`: 새로 삽입된 레코드 수
- `records_updated`: 업데이트된 레코드 수
- `success`: 동기화 성공 여부

### 3. Excel 파일 확인

**측정사업장 Excel 파일**에서 H0432 행을 찾아 다음을 확인:

1. **컬럼명 확인**: 첫 번째 행(헤더)에서 다음 컬럼명이 정확히 일치하는지 확인
   - "코드" (또는 "코드번호" 등 다른 이름인지)
   - "년도" (또는 "측정년도" 등)
   - "구분" (또는 "측정주기" 등)
   - "사업장명"

2. **데이터 확인**: H0432 행의 각 컬럼 값
   - 코드: "H0432"
   - 년도: 2026 (숫자)
   - 구분: "상반기" 또는 "하반기"
   - 사업장명: 비어있지 않음

### 4. 동기화 후 다시 확인

수동 동기화를 실행한 후, 아래 URL로 다시 확인:

```
https://html-tan-six.vercel.app/api/debug/h0432
```

**기대 결과:**
- `measurement_business.count`가 0에서 1 이상으로 증가
- `summary.has_in_measurement_business`가 false에서 true로 변경
