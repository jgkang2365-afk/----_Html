# 배포/재배포 가이드

## 파일 업로드 기능이 배포 사이트에 보이지 않는 경우

파일 업로드 기능은 코드에 구현되어 있지만, Git에 커밋되지 않아 배포되지 않았을 수 있습니다.

## 해결 방법

### 1. 변경사항 확인

```bash
git status
```

변경된 파일 확인:
- `app/dashboard/page.tsx` (ExcelUpload 컴포넌트 추가)
- `components/features/ExcelUpload.tsx` (새 파일)
- `app/api/upload/excel/route.ts` (새 파일)
- `lib/sync/excel-sync.ts` (Storage 지원 추가)

### 2. 파일 커밋 및 푸시

```bash
# 변경사항 스테이징
git add app/dashboard/page.tsx
git add components/features/ExcelUpload.tsx
git add app/api/upload/excel/route.ts
git add lib/sync/excel-sync.ts

# 커밋
git commit -m "파일 업로드 기능 및 Storage 동기화 지원 추가"

# 원격 저장소에 푸시
git push origin main
```

### 3. Vercel 자동 배포 확인

Git 푸시 후 Vercel이 자동으로 배포를 시작합니다:
1. Vercel 대시보드 접속
2. 프로젝트 → **Deployments** 탭 확인
3. 새로운 배포가 진행 중인지 확인
4. 배포 완료 대기 (약 2-3분)

### 4. 배포 완료 후 확인

1. 배포된 사이트 접속
2. 대시보드 페이지 확인
3. "Excel 파일 업로드" 섹션이 보이는지 확인

## 주의사항

### 커밋하지 말아야 할 파일

- `.env.local` (환경 변수)
- `node_modules/`
- `.next/`
- Excel 파일 (`.xls`, `.xlsx`)

이 파일들은 이미 `.gitignore`에 포함되어 있습니다.

### 배포 후에도 기능이 보이지 않는 경우

1. **브라우저 캐시 삭제**
   - Ctrl + Shift + Delete
   - 또는 하드 새로고침: Ctrl + F5

2. **빌드 로그 확인**
   - Vercel 대시보드 → Deployments → 최신 배포 → Build Logs
   - 빌드 오류가 있는지 확인

3. **컴포넌트 에러 확인**
   - 브라우저 개발자 도구 Console 탭 확인
   - 에러 메시지 확인

## 빠른 확인 명령어

```bash
# 변경사항 확인
git status

# 커밋할 파일만 확인
git status --short

# 커밋되지 않은 파일 목록
git diff --name-only
```
