# 환경 변수 설명

## NEXT_PUBLIC_SUPABASE_URL vs SUPABASE_URL

### 값은 동일합니다!

두 변수 모두 **같은 Supabase 프로젝트 URL**을 사용합니다.

```env
# 둘 다 같은 URL을 사용합니다
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
SUPABASE_URL=https://xxxxx.supabase.co  # ← 같은 값!
```

### 차이점: 접근 범위

| 변수명 | 접근 범위 | 용도 |
|--------|----------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | 클라이언트 사이드 (브라우저) | 브라우저에서 직접 접근 가능 |
| `SUPABASE_URL` | 서버 사이드만 | API Routes, Server Components에서만 사용 |

### 왜 두 개가 필요한가?

Next.js는 보안을 위해 환경 변수를 구분합니다:

1. **`NEXT_PUBLIC_` 접두사가 있는 변수**
   - 브라우저에 노출됨 (JavaScript 번들에 포함)
   - 클라이언트 컴포넌트에서 사용 가능
   - 예: `process.env.NEXT_PUBLIC_SUPABASE_URL`

2. **`NEXT_PUBLIC_` 접두사가 없는 변수**
   - 서버에서만 접근 가능
   - 브라우저에 노출되지 않음 (보안)
   - 예: `process.env.SUPABASE_URL`

## 실제 사용 예시

### 클라이언트 사이드 (브라우저)
```typescript
// components/ClientComponent.tsx
'use client';

import { createBrowserClient } from '@/lib/db/supabase';

// NEXT_PUBLIC_SUPABASE_URL 사용
const supabase = createBrowserClient();
```

### 서버 사이드 (API Routes)
```typescript
// app/api/something/route.ts
import { createServerClient } from '@/lib/db/supabase';

// SUPABASE_URL 사용 (서비스 롤 키와 함께)
const supabase = createServerClient();
```

## .env.local 파일 예시

```env
# Supabase 프로젝트 URL (같은 값 사용)
NEXT_PUBLIC_SUPABASE_URL=https://abcdefghijklmnop.supabase.co
SUPABASE_URL=https://abcdefghijklmnop.supabase.co

# 공개 키 (클라이언트에서 사용)
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6ImFiY2RlZmdoaWprbG1ub3AiLCJyb2xlIjoiYW5vbiIsImlhdCI6MTY0MTk3NjgwMCwiZXhwIjoxOTU3NTUyODAwfQ.xxxxx

# 서비스 롤 키 (서버에서만 사용, 절대 노출 금지!)
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6ImFiY2RlZmdoaWprbG1ub3AiLCJyb2xlIjoic2VydmljZV9yb2xlIiwiaWF0IjoxNjQxOTc2ODAwLCJleHAiOjE5NTc1NTI4MDB9.xxxxx
```

## 요약

- ✅ **URL은 동일**: `NEXT_PUBLIC_SUPABASE_URL`과 `SUPABASE_URL`은 같은 값
- ✅ **키는 다름**: `NEXT_PUBLIC_SUPABASE_ANON_KEY`와 `SUPABASE_SERVICE_ROLE_KEY`는 다른 값
- ✅ **보안**: `NEXT_PUBLIC_` 접두사가 없으면 서버에서만 접근 가능
