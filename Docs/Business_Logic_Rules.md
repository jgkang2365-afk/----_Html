# 주요 비즈니스 로직 및 용어 정의 (Business Logic Rules)

## 1. 최신 자료 (Latest Data)의 정의

시스템에서 사업장의 상태나 정보를 판단할 때 사용하는 "최신 자료"의 기준은 오직 `measurement_business` (측정사업장) 파일의 데이터에 기반하며, 다음의 우선순위를 따릅니다.

### 우선순위 결정 규칙 (Priority Rules)
데이터는 **년도(Year)** 와 **주기(Period)** 를 기준으로 정렬하여 가장 상위에 있는 데이터를 최신으로 간주합니다.

1.  **년도 (Year)**: 높은 년도가 우선 (통상 내림차순)
2.  **주기 (Period)**: 같은 년도 내에서는 **하반기**가 **상반기**보다 우선
    - (2분기, 4분기 등 하반기에 속하는 데이터 포함)

### 순서 예시 (Priority Example)
1.  2026년 상반기 (가장 최신)
2.  2025년 하반기
3.  2025년 상반기
4.  2024년 하반기
5.  2024년 상반기
...

> **Note**: 데이터 처리 속도 이슈가 발생할 경우, 검색 범위를 최근 1년 또는 2년으로 제한하는 것은 허용됩니다.

## 2. 데이터 정합성 검증 기준 (Data Verification Criteria)

데이터의 수정 및 검증은 외부의 개입 없이 오직 다음 두 파일(테이블)의 내용만을 기준으로 합니다.

1.  **`business_info` (사업장정보 - 현재 측정일지 관리 웹 상태)**
2.  **`measurement_business` (측정사업장 - 검증 기준)**

### 검증 예외 규칙 (Exclusion Rules)
- **사업장명에 "(번외)" 또는 "번외"가 포함된 경우**: 해당 데이터는 정합성 검증 대상에서 **제외**합니다.
    - 이는 검증이나 관리 대상이 아닌 별도 데이터로 간주합니다.

### 검증 로직
- `business_info`의 정보(사업장명, 대표자명 등)는 "최신 자료" 기준의 `measurement_business` 정보와 일치해야 합니다.
- **불일치 시 (Mismatch)**:
    - 정보가 다를 경우(사용자가 직접 수정한 정보가 더 최신이라 하더라도 기준 파일과 다르면) **"데이터 불일치"** 로그를 생성하고 지속적으로 표시합니다.
    - 이는 사용자에게 "기준 파일(MES DB)과 측정일지 관리 웹 정보가 다르니 확인하라"는 안내 역할을 합니다.
- **일치 시 (Match/Resolved)**:
    - 사용자가 정보를 수정하거나 MES DB 파일을 업데이트하여 두 정보가 일치하게 되면, 해당 **로그는 자동으로 삭제(갱신)** 됩니다.
