# 환경 변수 문제 해결 가이드

## 현재 상태
환경 변수가 로드되지 않고 있습니다. 다음을 확인하세요.

## 1. .env.local 파일 위치 확인
파일이 프로젝트 루트 디렉토리에 있어야 합니다:
```
측정일지_Html/
  ├── .env.local  ← 여기에 있어야 함
  ├── package.json
  ├── next.config.mjs
  └── app/
```

## 2. 파일 형식 확인

`.env.local` 파일은 다음 형식이어야 합니다:

```env
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6Inh4eHh4Iiwicm9sZSI6ImFub24iLCJpYXQiOjE2NDE5NzY4MDAsImV4cCI6MTk1NzU1MjgwMH0.xxxxx
SUPABASE_URL=https://xxxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6Inh4eHh4Iiwicm9sZSI6InNlcnZpY2Vfcm9sZSIsImlhdCI6MTY0MTk3NjgwMCwiZXhwIjoxOTU3NTUyODAwfQ.xxxxx
```

## 중요 사항

1. **따옴표 없이** 작성하세요
   - ❌ 잘못된 예: `NEXT_PUBLIC_SUPABASE_URL="https://..."` 
   - ✅ 올바른 예: `NEXT_PUBLIC_SUPABASE_URL=https://...`

2. **등호(=) 앞뒤에 공백 없이** 작성하세요
   - ❌ 잘못된 예: `NEXT_PUBLIC_SUPABASE_URL = https://...`
   - ✅ 올바른 예: `NEXT_PUBLIC_SUPABASE_URL=https://...`

3. **각 줄 끝에 공백이나 특수문자가 없는지** 확인하세요

4. **주석은 #으로 시작**하세요
   ```env
   # 이것은 주석입니다
   NEXT_PUBLIC_SUPABASE_URL=https://...
   ```

## 3. Supabase에서 값 가져오기

1. [Supabase 대시보드](https://supabase.com/dashboard) 접속
2. 프로젝트 선택
3. **Settings** > **API** 메뉴로 이동
4. 다음 정보 복사:
   - **Project URL** → `NEXT_PUBLIC_SUPABASE_URL`과 `SUPABASE_URL`에 동일하게 사용
   - **anon public** 키 → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - **service_role** 키 → `SUPABASE_SERVICE_ROLE_KEY` (⚠️ 주의: 이 키는 비밀입니다!)

## 4. 파일 저장 후 확인

1. `.env.local` 파일 저장
2. 개발 서버 **완전히 종료** (Ctrl+C)
3. 다시 시작: `npm run dev`
4. 브라우저에서 테스트: `http://localhost:3000/api/test-db`

## 5. 빠른 확인 방법

터미널에서 다음 명령어로 확인:
```bash
node check-env.js
```

모든 항목이 "✓ 설정됨"으로 표시되어야 합니다.

## 문제가 계속되면

1. 파일 인코딩 확인: UTF-8로 저장되어 있는지 확인
2. 파일명 확인: `.env.local` (숨김 파일일 수 있음)
3. 프로젝트 루트 확인: `package.json`과 같은 디렉토리에 있는지 확인
