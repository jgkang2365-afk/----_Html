# Vercel 배포 환경에서 Excel 동기화 문제

## 문제 상황

**Vercel에 배포된 환경에서 수동 동기화를 실행했는데 데이터베이스에 데이터가 누락되는 문제**

## 주요 원인

### 1. Vercel은 서버리스 환경

Vercel은 **서버리스(Serverless) 환경**입니다. 이는:
- 파일 시스템이 읽기 전용 (빌드 시점에만 접근 가능)
- 런타임에 프로젝트 폴더에 파일을 저장하거나 읽을 수 없음
- Excel 파일이 프로젝트 폴더에 있어도 배포 후에는 접근 불가

### 2. 현재 코드의 파일 경로

현재 동기화 코드는 다음 경로에서 Excel 파일을 찾습니다:

```typescript
const defaultPathXlsx = join(process.cwd(), "사업장정보.xlsx");
const defaultPathXls = join(process.cwd(), "사업장정보.xls");
```

**문제:** Vercel 배포 환경에서는 `process.cwd()`가 빌드된 코드 위치를 가리키며, Excel 파일은 포함되지 않습니다.

## 해결 방법

### 방법 1: 파일 업로드 기능 구현 (권장)

**장점:**
- Excel 파일을 Supabase Storage나 클라우드 저장소에 저장
- 배포 환경에서도 접근 가능
- 사용자가 웹 인터페이스에서 파일 업로드 가능

**구현 필요:**
1. 파일 업로드 API (`/api/upload/excel`)
2. Supabase Storage 연동
3. 업로드된 파일 경로로 동기화 실행

### 방법 2: 외부 파일 서버 사용

**장점:**
- Excel 파일을 별도 서버에 저장
- HTTP로 파일 접근 가능

**단점:**
- 추가 인프라 필요
- 파일 서버 관리 필요

### 방법 3: 로컬 서버에서 실행 (임시 방안)

**장점:**
- 현재 코드 그대로 사용 가능
- 빠른 해결

**단점:**
- Vercel의 장점(자동 스케일링, 글로벌 CDN)을 활용하지 못함
- 서버 유지 관리 필요

## 현재 상태 확인

### 1. 동기화 로그 확인

대시보드 → "Excel 파일 동기화 상태" 섹션에서:
- 동기화 상태 (성공/실패)
- 오류 메시지 확인
- 처리된 레코드 수 확인

### 2. 예상 오류 메시지

다음과 같은 오류가 발생할 수 있습니다:

```
Excel 파일을 찾을 수 없습니다: 사업장정보.xlsx 또는 사업장정보.xls
```

### 3. 개발 환경과 프로덕션 환경 차이

| 환경 | Excel 파일 위치 | 접근 가능 여부 |
|------|----------------|---------------|
| 로컬 개발 | 프로젝트 루트 | ✅ 가능 |
| Vercel 배포 | 없음 | ❌ 불가능 |

## 즉시 확인 사항

1. **대시보드에서 동기화 로그 확인:**
   - 동기화 상태가 "실패"인지 확인
   - 오류 메시지 확인

2. **브라우저 개발자 도구 확인:**
   - F12 → Console 탭
   - Network 탭에서 `/api/sync` 요청 확인
   - 오류 응답 확인

3. **Vercel 로그 확인:**
   - Vercel 대시보드 → 프로젝트 → Functions → Logs
   - 동기화 시도 시 오류 메시지 확인

## 임시 해결 방법 (로컬 개발 환경)

로컬 개발 환경에서는 정상 작동합니다:

1. 프로젝트 루트에 Excel 파일 배치:
   ```
   측정일지_Html/
     ├── 사업장정보.xlsx
     ├── 측정사업장.xlsx
     └── ...
   ```

2. 로컬 개발 서버 실행:
   ```bash
   npm run dev
   ```

3. 로컬에서 동기화 테스트:
   - `http://localhost:3000/dashboard` 접속
   - "수동 동기화" 버튼 클릭

## 장기 해결 방법

파일 업로드 기능을 구현하여 Vercel 배포 환경에서도 동기화가 가능하도록 해야 합니다.

### 구현 계획

1. **Supabase Storage 설정**
   - Storage 버킷 생성 (예: `excel-files`)
   - 업로드 권한 설정

2. **파일 업로드 API 구현**
   - `/api/upload/excel` 엔드포인트
   - Supabase Storage에 파일 저장

3. **동기화 로직 수정**
   - Storage에서 파일 다운로드
   - 메모리에서 파싱
   - 동기화 실행

4. **UI 구현**
   - 파일 업로드 컴포넌트
   - 업로드 진행 상태 표시

## 결론

**현재 Vercel 배포 환경에서는 Excel 파일에 접근할 수 없어 동기화가 실패합니다.**

임시로는 로컬 개발 환경에서 테스트하거나, 파일 업로드 기능을 구현해야 합니다.
