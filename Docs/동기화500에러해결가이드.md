# 동기화 500 에러 해결 가이드

## 문제 상황

수동 동기화 실행 시 다음과 같은 오류가 발생합니다:
- 파싱은 성공 (데이터 파싱 완료)
- `POST /api/sync 500` 에러 발생
- 처리 시간: 약 5-6초

## 에러 확인 방법

### 1. 브라우저 개발자 도구 확인

1. **F12** 키를 눌러 개발자 도구 열기
2. **Console** 탭 확인:
   - 빨간색 에러 메시지 확인
   - JavaScript 에러 메시지 확인
3. **Network** 탭 확인:
   - `/api/sync` 요청 클릭
   - **Response** 탭에서 상세 에러 메시지 확인

### 2. 대시보드에서 동기화 로그 확인

1. 대시보드 페이지 접속
2. "Excel 파일 동기화 상태" 섹션 확인
3. "실패" 상태인 동기화 로그 클릭
4. 오류 메시지 확인

### 3. 서버 터미널 확인

개발 서버가 실행 중인 터미널에서:
- 에러 스택 트레이스 확인
- `console.error` 출력 확인

## 가능한 원인

### 1. 데이터베이스 제약조건 위반

**증상:**
- 데이터 타입 불일치
- NOT NULL 제약조건 위반
- UNIQUE 제약조건 위반

**해결 방법:**
- 에러 메시지에서 어떤 필드가 문제인지 확인
- Excel 파일의 데이터 형식 확인
- 데이터베이스 스키마 확인

### 2. 네트워크 드라이브 접근 문제

**증상:**
- Z: 드라이브 접근 불가
- 파일 읽기 권한 없음
- 네트워크 연결 끊김

**해결 방법:**
- Z: 드라이브가 마운트되어 있는지 확인
- 파일 경로 접근 권한 확인
- 파일 탐색기에서 직접 접근 테스트

### 3. 데이터 크기 문제

**증상:**
- 대용량 데이터 처리 중 타임아웃
- 메모리 부족

**해결 방법:**
- 배치 크기 조정 (코드 수정 필요)
- 데이터베이스 연결 타임아웃 설정 확인

### 4. 필수 필드 누락

**증상:**
- NOT NULL 필드에 null 값 삽입 시도

**해결 방법:**
- Excel 파일에서 필수 필드(코드, 사업장명) 확인
- 파싱 로직에서 필수 필드 검증 확인

## 즉시 확인 사항

1. **브라우저 개발자 도구에서 에러 메시지 확인**
   - F12 → Console 탭
   - F12 → Network 탭 → `/api/sync` → Response

2. **대시보드에서 동기화 로그 확인**
   - 동기화 상태가 "실패"인 항목의 오류 메시지 확인

3. **파일 접근 확인**
   - 파일 탐색기에서 `Z:\data\측정팀\자동화 툴\MES 프로그램 DB` 접근 확인
   - `사업장정보.xlsx` 파일 존재 및 읽기 권한 확인

## 다음 단계

상세 에러 메시지를 확인한 후:
1. 에러 메시지의 내용을 공유해주시면 더 구체적인 해결 방법을 제시하겠습니다.
2. 특히 데이터베이스 에러 메시지 (예: "column ... does not exist", "null value in column ...")를 확인해주세요.
