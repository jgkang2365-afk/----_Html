# 환경 변수 체크리스트

`.env.local` 파일에 다음 4개의 환경 변수가 모두 설정되어 있어야 합니다:

## 필수 환경 변수

```env
# 클라이언트 사이드용 (NEXT_PUBLIC_ 접두사 필요)
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# 서버 사이드용 (NEXT_PUBLIC_ 접두사 없음)
SUPABASE_URL=https://xxxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## 확인 사항

- [ ] `.env.local` 파일이 프로젝트 루트 디렉토리에 있는가?
- [ ] 파일명이 정확히 `.env.local`인가? (`.env.local.txt` 아님)
- [ ] 4개의 환경 변수가 모두 있는가?
- [ ] `NEXT_PUBLIC_` 접두사가 올바르게 붙어 있는가?
- [ ] 값에 따옴표나 공백이 없는가?
- [ ] Supabase 프로젝트 URL이 `https://`로 시작하는가?

## 테스트 방법

1. 개발 서버 재시작 (중요!)
   ```bash
   # 서버 중지 (Ctrl+C)
   npm run dev
   ```

2. 브라우저에서 테스트:
   ```
   http://localhost:3000/api/test-db
   ```

3. 성공 응답 예시:
   ```json
   {
     "success": true,
     "message": "데이터베이스 연결 성공",
     "envCheck": {
       "hasSupabaseUrl": true,
       "hasServiceRoleKey": true,
       "hasPublicUrl": true,
       "hasPublicKey": true
     }
   }
   ```

## 문제 해결

### envCheck가 모두 false인 경우
- `.env.local` 파일이 프로젝트 루트에 있는지 확인
- 개발 서버를 재시작했는지 확인
- 파일명이 정확한지 확인

### 일부만 true인 경우
- 누락된 환경 변수를 추가
- 변수명 철자 확인
- `NEXT_PUBLIC_` 접두사 확인
