# 로컬 환경에서 H0432 테스트 가이드

## 목표

로컬 환경에서 H0432 데이터가 정상적으로 동기화되고 검색되는지 확인하여, 배포 환경과의 차이를 파악합니다.

---

## 사전 준비사항

### 1. 환경 변수 확인

로컬 환경에서 `.env.local` 파일이 있어야 합니다:

```env
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
SUPABASE_URL=https://xxxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

**위치**: 프로젝트 루트 디렉토리 (`측정일지_Html/.env.local`)

### 2. 최신 Excel 파일 확인

H0432가 포함된 최신 `측정사업장.xlsx` 파일을 준비하세요.

---

## 단계별 테스트 절차

### 1단계: 개발 서버 실행

터미널에서 다음 명령어 실행:

```bash
# 프로젝트 루트 디렉토리로 이동 (이미 있다면 생략)
cd C:\Users\USER\Desktop\cursor\측정일지_Html

# 개발 서버 실행
npm run dev
```

**예상 출력:**
```
▲ Next.js 14.2.0
- Local:        http://localhost:3000
- Ready in 2.5s
```

서버가 실행되면 브라우저에서 `http://localhost:3000` 접속 가능합니다.

---

### 2단계: 로그인 및 대시보드 접속

1. **브라우저에서 접속**
   - http://localhost:3000
   - 로그인 페이지가 나타나면 로그인

2. **대시보드 이동**
   - 로그인 후 대시보드 (`/dashboard`) 접속
   - 또는 직접: http://localhost:3000/dashboard

---

### 3단계: 최신 Excel 파일 업로드

**중요**: 로컬 테스트이지만, 코드가 Supabase Storage에서 파일을 읽도록 되어 있으므로 Storage에 파일을 업로드해야 합니다.

1. **대시보드에서 파일 업로드 섹션 찾기**
   - Excel 파일 업로드 컴포넌트 확인

2. **H0432가 포함된 최신 파일 업로드**
   - `측정사업장.xlsx` 파일 선택
   - 업로드 버튼 클릭
   - 업로드 성공 메시지 확인

3. **업로드된 파일 확인**
   - 파일명과 업로드 시간 확인
   - H0432가 포함된 최신 버전인지 재확인

---

### 4단계: 로컬에서 동기화 실행

#### 방법 A: 대시보드에서 수동 동기화 (권장)

1. **대시보드에서 "수동 동기화" 버튼 클릭**
   - 또는 "측정사업장" 파일만 동기화 버튼 (있는 경우)

2. **동기화 진행 대기**
   - 성공/실패 메시지 확인

3. **터미널 로그 확인**
   - 개발 서버가 실행 중인 터미널 창 확인
   - 다음과 같은 로그를 찾아보세요:
     - `[측정사업장 동기화] Excel 파일에서 읽은 데이터 행 수: XXXX`
     - `[측정사업장 동기화] 파싱된 데이터 중 H0432 포함: X건`
     - `[측정사업장 동기화] '코드' 컬럼의 H로 시작하는 코드 샘플`

#### 방법 B: API 직접 호출 (고급)

브라우저 개발자 도구 (F12) → Console 탭에서:

```javascript
// 측정사업장 파일만 동기화
fetch('/api/sync?type=measurement-business', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  }
})
.then(res => res.json())
.then(data => console.log('동기화 결과:', data))
.catch(err => console.error('오류:', err));
```

### 5단계: 터미널 로그 분석

개발 서버 터미널에서 다음 로그를 확인하세요:

#### ✅ 성공 시나리오

```
[측정사업장 동기화] Excel 파일에서 읽은 데이터 행 수: 1486

[측정사업장 동기화] '코드' 컬럼의 H로 시작하는 코드 샘플: [ ..., 'H0432', ... ]
[측정사업장 동기화] 파싱된 데이터 중 H0432 포함: 1건 (또는 그 이상)

#### ❌ 실패 시나리오 (현재 배포 환경 상황)

```

**중요**: 로그에서 H0432가 포함되어 있는지, 샘플 목록에 나타나는지 확인하세요.

### 6단계: H0432 검색 테스트

#### 방법 A: 브라우저에서 검색

1. **측정일지 페이지 접속**
   - http://localhost:3000/journal

2. **검색 조건 입력**
   - 코드: `H0432`
   - 또는 사업장명으로 검색 (H0432와 연관된 사업장명)

3. **검색 결과 확인**
   - H0432 데이터가 나타나는지 확인

#### 방법 B: API 직접 테스트

브라우저 개발자 도구 (F12) → Console 탭에서:

```javascript
// H0432 검색
fetch('/api/journal/search?code=H0432')
  .then(res => res.json())
  .then(data => {
    console.log('검색 결과:', data);
    if (data.results && data.results.length > 0) {
      console.log('✅ H0432 데이터 발견!', data.results);
    } else {
      console.log('❌ H0432 데이터 없음');
    }
  })
  .catch(err => console.error('오류:', err));
```

## 로그 확인 포인트

터미널 로그에서 다음 정보를 확인하세요:

### 1. Excel 파일 읽기 성공 여부
```
[측정사업장 동기화] Excel 파일에서 읽은 데이터 행 수: XXXX
```
- 행 수가 0이면 파일 읽기 실패

### 2. 코드 컬럼 발견 여부
```
[측정사업장 동기화] "코드" 컬럼 (정확히 일치): 코드
```
- "코드" 컬럼을 찾지 못하면 컬럼명 문제

### 3. H0432 포함 여부
```
[측정사업장 동기화] 파싱된 데이터 중 H0432 포함: X건
```
- 0건이면 H0432가 파일에 없거나 파싱 실패

### 4. 코드 샘플 목록
```
[측정사업장 동기화] '코드' 컬럼의 H로 시작하는 코드 샘플: [ ... ]
```
- 샘플 목록에 H0432가 있으면 파일에는 있지만 파싱에서 누락
- 샘플 목록에 H0432가 없으면 파일 자체에 없음

## 문제 해결

### 문제 1: 개발 서버가 실행되지 않음

**해결:**
```bash
# 의존성 설치
npm install

# 다시 시도
npm run dev
```

### 문제 2: 로그인 실패

**해결:**
- Supabase 대시보드에서 사용자 계정 확인
- `.env.local` 파일의 환경 변수 확인

### 문제 3: 파일 업로드 실패

**해결:**
- Supabase Storage 권한 확인
- 파일 크기 제한 확인 (일반적으로 50MB)
- 파일 형식 확인 (.xlsx 권장)

### 문제 4: 동기화 실패

**해결:**
- 터미널 로그의 오류 메시지 확인
- Supabase 연결 상태 확인
- 데이터베이스 권한 확인
