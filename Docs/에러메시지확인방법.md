# 에러 메시지 확인 방법

## 현재 상황

브라우저 콘솔에 다음 메시지가 표시됩니다:
```
동기화 오류 상세: Object
```

하지만 Object의 내용이 펼쳐지지 않아 실제 에러 메시지를 확인할 수 없습니다.

## 에러 메시지 확인 방법

### 방법 1: 콘솔에서 Object 펼쳐보기 (가장 쉬움)

1. **F12** 키를 눌러 개발자 도구 열기
2. **Console** 탭으로 이동
3. `동기화 오류 상세: Object` 메시지 찾기
4. Object 앞의 **▶ (화살표)** 클릭하여 펼치기
5. 펼쳐진 내용에서 다음 정보 확인:
   - `error`: 에러 메시지
   - `details`: 스택 트레이스 (있다면)

### 방법 2: Network 탭에서 Response 확인 (더 상세함)

1. **F12** 키를 눌러 개발자 도구 열기
2. **Network** 탭으로 이동
3. **필터**: `sync` 입력하여 `/api/sync` 요청만 표시
4. **수동 동기화 버튼 클릭** (또는 페이지 새로고침)
5. `/api/sync` 요청 클릭
6. **Response** 탭 클릭
7. 다음 형식의 JSON 응답 확인:
   ```json
   {
     "success": false,
     "error": "여기에 에러 메시지가 있습니다",
     "details": "스택 트레이스 (있다면)"
   }
   ```

### 방법 3: 콘솔에서 직접 확인

콘솔에서 다음 명령어를 입력하여 마지막 동기화 에러 확인:
```javascript
// 콘솔에서 실행
fetch('/api/sync', { method: 'POST' })
  .then(r => r.json())
  .then(data => console.log('동기화 응답:', data))
  .catch(err => console.error('에러:', err));
```

## 예상되는 에러 메시지 유형

### 1. 파일 접근 에러
```
Excel 파일을 찾을 수 없습니다: 사업장정보.xlsx 또는 사업장정보.xls
```
**해결**: Z: 드라이브 접근 확인, 파일 경로 확인

### 2. 데이터베이스 에러
```
column "xxx" does not exist
```
**해결**: 데이터베이스 마이그레이션 확인

### 3. 데이터 타입 에러
```
null value in column "xxx" violates not-null constraint
```
**해결**: Excel 파일의 필수 필드 확인

### 4. 네트워크 드라이브 에러
```
ENOENT: no such file or directory
```
**해결**: Z: 드라이브 마운트 확인

## 다음 단계

에러 메시지를 확인한 후:
1. 에러 메시지의 전체 내용을 복사하여 공유해주세요
2. 특히 `error` 필드의 메시지를 확인해주세요
3. `details` 필드에 스택 트레이스가 있다면 그것도 확인해주세요

에러 메시지를 알려주시면 정확한 해결 방법을 제시하겠습니다.
