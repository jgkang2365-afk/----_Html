# 사업장 정보 누락 문제 해결 가이드

## 문제 원인

엑셀 동기화 시 사업장 정보가 누락되는 이유는 **필수 필드 체크** 때문입니다.

`lib/sync/excel-sync.ts`의 `parseBusinessInfo` 함수에서 다음 필터가 적용됩니다:

```typescript
}).filter((row) => row.code && row.business_name); // 필수 필드 체크
```

**다음 경우에 행이 필터링되어 제외됩니다:**
1. "코드" 컬럼이 비어있는 경우
2. "사업장명" 컬럼이 비어있는 경우
3. Excel 파일의 컬럼명이 정확히 일치하지 않는 경우

## 확인 사항

### 1. Excel 파일 컬럼명 확인

Excel 파일의 첫 번째 행(헤더)에 다음 컬럼명이 정확히 있는지 확인:
- **코드** (정확히 "코드"여야 함)
- **사업장명** (정확히 "사업장명"이어야 함)

**주의:**
- 공백이 있으면 안 됩니다 (예: "코드 ", " 코드" ❌)
- 대소문자 구분 (보통 대소문자 상관없지만 확인 필요)

### 2. Excel 파일 구조 확인 방법

브라우저에서 다음 URL로 접속하여 Excel 파일 구조를 확인할 수 있습니다:

```
https://your-site.vercel.app/api/test-excel?file=business-info
```

또는 로컬에서:
```
http://localhost:3000/api/test-excel?file=business-info
```

이 API는 다음을 보여줍니다:
- Excel 파일의 컬럼명 목록
- 첫 번째 데이터 행 샘플
- 총 행 수, 총 컬럼 수

### 3. 동기화 로그 확인

대시보드에서 동기화 로그를 확인하여:
- `records_processed`: 실제 처리된 행 수 (필터링 후)
- Excel 파일의 전체 행 수와 비교

만약 Excel 파일에 100개 행이 있는데 `records_processed`가 50개라면, 50개 행이 필터링된 것입니다.

## 해결 방법

### 방법 1: Excel 파일 수정 (권장)

1. Excel 파일 열기
2. 첫 번째 행(헤더) 확인:
   - "코드" 컬럼이 있는지 확인
   - "사업장명" 컬럼이 있는지 확인
3. 데이터 행 확인:
   - "코드" 컬럼에 값이 있는 행만 남기기
   - "사업장명" 컬럼에 값이 있는 행만 남기기
   - 비어있는 행 삭제

### 방법 2: 컬럼명 확인 및 수정

Excel 파일의 컬럼명이 다르다면:
- Excel 파일에서 컬럼명을 "코드", "사업장명"으로 변경
- 또는 코드에서 컬럼명 매핑 수정 (고급 사용자용)

### 방법 3: 필터링 로직 확인

필터링된 행이 많은 경우, Excel 파일에서 다음을 확인:
1. 빈 행이 있는지
2. 코드나 사업장명이 없는 행이 있는지
3. 헤더 행이 올바른지 (첫 번째 행이 헤더인지)

## 디버깅 팁

### 1. Excel 파일 읽기 테스트

```bash
# 로컬 개발 환경에서
curl http://localhost:3000/api/test-excel?file=business-info
```

### 2. 동기화 실행 후 로그 확인

동기화를 실행한 후:
1. 대시보드 → 동기화 로그 확인
2. `records_processed` 값 확인
3. Excel 파일의 행 수와 비교

### 3. 필터링된 행 확인 (개발 환경)

개발 환경에서 `lib/sync/excel-sync.ts`의 `parseBusinessInfo` 함수에 로깅 추가:

```typescript
const parsedData = parseBusinessInfo(excelData);
console.log(`원본 데이터: ${excelData.length}행`);
console.log(`파싱 후: ${parsedData.length}행`);
console.log(`필터링됨: ${excelData.length - parsedData.length}행`);
```

## 예상 시나리오

### 시나리오 1: Excel 파일에 빈 행이 있는 경우

**문제:** Excel 파일 중간에 빈 행이 있으면 해당 행이 필터링됨

**해결:** Excel 파일에서 빈 행 삭제

### 시나리오 2: 컬럼명이 다른 경우

**문제:** Excel 파일의 컬럼명이 "사업장명"이 아니라 "사업장 이름" 등으로 되어 있음

**해결:** Excel 파일에서 컬럼명을 "코드", "사업장명"으로 변경

### 시나리오 3: 코드나 사업장명이 비어있는 경우

**문제:** 일부 행에 코드나 사업장명이 비어있음

**해결:** 
- 해당 데이터를 채우거나
- 비어있는 행은 제외하는 것이 정상 동작입니다 (의도된 동작)

## 추가 확인 사항

1. **Vercel 배포 환경:** 
   - Vercel은 서버리스 환경이므로 로컬 파일 시스템에 Excel 파일을 저장할 수 없습니다
   - Excel 파일이 Vercel 환경에서 접근 가능한 위치에 있어야 합니다
   - 현재 코드는 `process.cwd()` (프로젝트 루트)에서 파일을 찾습니다

2. **파일 경로:**
   - 로컬: 프로젝트 루트에 `사업장정보.xlsx` 또는 `사업장정보.xls` 파일이 있어야 함
   - Vercel: 현재는 파일 시스템 접근 불가 (향후 개선 필요)

## 결론

사업장 정보 누락은 **의도된 동작**입니다. 코드나 사업장명이 비어있는 행은 필수 필드 체크로 인해 자동으로 제외됩니다.

만약 Excel 파일에 모든 데이터가 있는데도 누락된다면:
1. Excel 파일의 컬럼명 확인
2. `/api/test-excel` API로 파일 구조 확인
3. 동기화 로그에서 처리된 행 수 확인
