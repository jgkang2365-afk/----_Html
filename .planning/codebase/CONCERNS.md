# Concerns & Technical Debt - 측정일지 관리 시스템

시스템의 안정성을 저해하거나 향후 개선이 필요한 기술적 우려 사항 및 부채 목록입니다.

## 1. 엑셀 파싱의 취약성 (Brittle Excel Parsing)
- **우려 사항**: `excel-sync.ts`에서 특정 필드(예: BK열)를 하드코딩된 인덱스로 참조하고 있음.
- **위험**: 엑셀 양식이 변경될 경우 데이터가 잘못 매핑되거나 런타임 에러 발생 가능.
- **해결 방안**: 헤더 이름을 기반으로 동적 매핑 로직을 강화하거나 엑셀 템플릿 버전 관리 도입 필요.

## 2. 루트 디렉토리 클러터 (Root Clutter)
- **상태**: 다수의 `tmp_`, `check_` 파일들이 루트에 산재하여 프로젝트 가독성 저하.
- **조치**: 현재 GSD2 워크플로우를 통해 대대적인 정화 작업 진행 중.

## 3. OS 의존성 (OS Dependency)
- **내용**: K2B 자동화 시 윈도우의 `SendKeys`를 활용하기 위해 PowerShell 명령에 의존함.
- **위험**: 윈도우 외의 환경(Linux/Docker)에서는 작동하지 않으며, 서버 성능에 따라 타이밍 이슈(Wait Time) 발생 가능.

## 4. 데이터 동기화 및 로직 무결성 (Logical Integrity) - [NEW]
- **연번 중복 위험**: `lib/utils/number-assignment.ts`에서 최대값을 조회 후 JS에서 +1 하는 방식은 원자성이 보장되지 않으므로 다중 사용자 환경에서 연번이 중복될 수 있음.
- **인원 변동 대응 부재**: 이미 연번이 부여된 이후 `total_employees`가 변경(4인 -> 5인 이상)되어도 연번이 자동으로 재할당되지 않아 보고서 상의 무결성이 깨질 위험이 있음.
- **거대 API (Mega-Fetch)**: `api/journal/search/route.ts`에서 5개 테이블을 JS 레벨에서 병합(Join)하고 있어 데이터 1만 건 이상 시 심각한 성능 저하 우려.
