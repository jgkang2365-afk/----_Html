# Technical Concerns - 측정일지 관리 시스템

시스템의 품질 향상을 위해 향후 개선이 필요하거나 현재 모니터링 중인 이슈들입니다.

## 1. 엑셀 동기화 엔진의 파편화 이슈
- **내용**: `lib/sync/excel-sync.ts` 파일이 2,300라인을 넘어가며 너무 무거워짐.
- **영향**: 코드 가독성 저하 및 수정 시 사이드 이펙트 우려.
- **해석**: 사업장 동기화, 파싱, 보정 로직을 별도 모듈로 분리(Refactoring)하는 작업이 필요.

## 2. 테스트 자동화의 부재
- **내용**: 현재 유닛 테스트(Unit Test)를 위한 프레임워크가 부재하며 수동 스크립트에 의존함.
- **영향**: 핵심 알고리즘 수정 시 회귀 오류 발견이 늦어질 수 있음.
- **우선순위: 중**

## 3. K2B 자동화의 불안정성
- **내용**: 외부 전산망의 UI 변경 시 Selenium 로직이 파손될 가능성이 상시 존재.
- **대응**: 예외 처리 강화 및 UI 변경 감지 알림 로직 검토 중.

## 4. 데이터 히스토리 보존 (Snapshot)
- **내용**: 현재 'Latest Wins' 정책으로 인해 기존 데이터가 엑셀 데이터로 덮어써질 때, 이전 값의 이력이 남지 않음.
- **대응**: `sync_history_snapshots`와 같은 테이블을 추가하여 덮어쓰기 전 데이터를 보존하는 방안 검토 필요.
