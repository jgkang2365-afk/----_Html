# Development Conventions & Core Logic - 측정일지 관리 시스템

시스템의 일관성을 유지하기 위해 정의된 규약과 성공적으로 운영 중인 핵심 비즈니스 로직(The Joo Rule)을 기록합니다.

## 1. 성공 DNA: 핵심 알고리즘 및 정규표현식

### 1-1. 이메일 추출 정규표현식
측정사업장 엑셀 파일에서 담당자 또는 계산서 이메일을 추출할 때 사용되는 패턴입니다.
```regex
/[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}/g
```
- **구현 위치**: `lib/sync/excel-sync.ts` 내 `parseMeasurementBusiness` 함수.

### 1-2. 5인 이상 연번 부여 알고리즘
사업장의 총인원에 따라 연번을 순차 부여하되, 특정 조건을 만족할 때만 증가시키는 핵심 정책입니다.
- **규칙**: 5인 이상일 때만 번호를 증가시키고, 5인 미만일 경우 직전에 부여된 번호를 그대로 사용(중복 허용).
- **예시**: A(3명)-000, B(8명)-001, C(5명)-002, D(4명)-002
- **구현 위치**: `lib/utils/number-assignment.ts`

### 1-3. 데이터 정제 (Normalization)
- **전화번호/사업자번호**: 공백 및 하이픈 제거 후 숫자만 추출.
- **산재관리번호**: 11자리 보정 (10자리인 경우 마지막에 '0' 추가 등).
```typescript
const normalizeDigits = (str: string) => str.replace(/[^0-9]/g, '');
```

## 2. 엑셀 파싱 컨벤션

### 2-1. 하드코딩된 컬럼 인덱스 (Column Indexing)
엑셀 라이브러리로 헤더를 읽지 못할 경우를 대비해 고정된 인덱스를 사용하며, 이는 현재 시스템에서 가장 안정적으로 작동하는 방식입니다.
- **Email/Mobile**: BK열(62, 63번 인덱스) 근처에서 담당자 정보를 추출.

### 2-2. 날짜 처리
- `excelDateToJSDate`: 엑셀 고유의 숫자형 날짜와 `YYYYMMDD`, `YYYY-MM-DD` 문자열 형식을 모두 지원하도록 범용 함수를 사용함.

## 3. 코드 스타일 가이드
- **기능 분리**: 하나의 함수는 하나의 엑셀 시트 또는 하나의 API 책임만 가짐.
- **KS 시간대**: 모든 시간 처리는 한국 표준시(KST)를 기준으로 `getKSTISOString()` 유틸리티를 통해 처리함.
