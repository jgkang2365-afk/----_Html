# Testing Strategy - 측정일지 관리 시스템

본 시스템의 품질 보증을 위한 테스트 전략 및 방법론을 정의합니다.

## 1. Selenium 기반 자동화 테스트
- **대상**: K2B 사이트 전송 로직.
- **방식**: 실제 브라우저를 띄워 로그인 및 데이터 입력 프로세스를 시뮬레이션함.
- **도구**: `Selenium WebDriver`, `ChromeDriver`.
- **관련 파일**: `lib/automation/k2b-service.ts` 내의 테스트 메서드들.

## 2. API & 비즈니스 로직 테스트
- **대상**: 엑셀 파싱 및 연번 부여 알고리즘.
- **방식**: `tmp_check_api.ts` 등 임시 스크립트를 활용한 단위 기능 검증 (현재는 주로 수동 실행 방식).
- **개선 방향**: 향후 Jest나 Vitest를 이용한 자동 단위 테스트 환경 구축 필요.

## 3. 데이터 정합성 검사 (Excel vs DB)
- **방식**: 엑셀 파일 업로드 후 DB에 저장된 레코드 수와 필드 값을 엑셀 원본과 비교하는 `Sync Log` 대조.
- **도구**: Supabase Dashboard 및 `sync_log` 테이블.

## 4. 수동 검증 포인트 (Manual QA)
- 5인 미만 사업장 처리 시 연번이 중복 부여되는지 확인.
- 관할 지청 매칭 로직이 주소에 따라 정확히 분류되는지 확인.
