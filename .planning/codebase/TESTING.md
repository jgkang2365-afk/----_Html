# Testing Strategy - 측정일지 관리 시스템

시스템의 신뢰성을 보장하기 위한 테스트 레이어와 검증 방법론입니다.

## 1. Selenium 기반 자동화 테스트
- **대상**: K2B 전산망 데이터 송신 및 파일 업로드.
- **방식**: 실제 브라우저를 구동하여 로그인 -> 메뉴 진입 -> 데이터 입력 -> 성적서 업로드 과정을 시뮬레이션.
- **검증**: `lib/automation/k2b-service.ts` 내의 개별 스텝별 성공여부 로그 확인.

## 2. 데이터 무결성 검증 (Self-Check Scripts)
- **방식**: `tmp/` 폴더 내의 일회성 스크립트를 사용하여 복잡한 알고리즘(5인 이상 연번, 영업일 계산 등)의 결과값이 기대치와 일치하는지 수동 검증.
- **주요 도구**: `ts-node scripts/verify-business-logic.ts`
- **검증 항목**: 지청별 예외 로직 적용 여부, 'Latest Wins' 덮어쓰기 정상 작동 여부.

## 3. 회귀 테스트 (The Joo Rule 준수)
- **필수 체크리스트**: 신규 기능 배포 전 반드시 컨벤션 문서의 '성공 DNA' 항목(이메일 추출 정규식 등)을 대조하여 기존 기능이 파손되지 않았는지 확인.

## 4. UI/UX 정적 검증
- **방식**: `JournalEditForm` 등 핵심 폼에서 데이터 입력 시 측정일수가 비즈니스 캘린더 기준(공휴일 제외)으로 자동 계산되는지 즉시 피드백 확인.
