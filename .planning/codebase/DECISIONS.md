# Design Decisions - 측정일지 관리 시스템

프로젝트의 방향을 좌우한 주요 설계 결정 사항들을 기록합니다.

## 1. Next.js 14 & Supabase 도입 (2025-Q1)
- **배경**: 기존 엑셀 기반 관리의 한계로 인해, 다수 사용자가 동시 접속 가능하고 직관적인 UI를 제공하는 시스템 필요.
- **결정**: 효율적인 서버 사이드 렌더링(SSR)과 빠른 DB 구축을 위해 Next.js + Supabase 페어 채택.

## 2. Selenium 기반 로우 레벨 자동화 (2025-Q2)
- **배경**: 고용노동부(K2B) 시스템이 API를 지원하지 않으며, 복잡한 보안 모듈과 동적 UI로 구성됨.
- **결정**: 현실적인 자동화 대안으로 `Selenium WebDriver`를 사용한 웹 자동화 구현. 일부 윈도우 한정 보안창은 PowerShell `SendKeys`로 보정.

## 3. 5인 미만 사업장 연번 중복 허용 (2025-Q4)
- **배경**: 특정 노동지청의 산정 기준에 따라 소규모 사업장은 번호를 공유하는 관행 존재.
- **결정**: `assignFivePlusSequenceNumber` 알고리즘을 통해 5인 미만 시 이전 번호를 재사용하도록 비즈니스 로직 고착화.

## 4. 멀티 데이트 지원 및 측정일수 자동화 (2026-04)
- **배경**: 불연속적인 3일 이상의 측정 건 처리를 수기 계산하던 방식에서 오류 다수 발생.
- **결정**: 측정일지 테이블에 `measurement_days` 필드를 추가하고, 공휴일 캘린더를 참조하여 실 측정일수를 자동 계산하는 기능 도입.

## 5. 'Latest Wins' 동기화 정책 (2026-04-11)
- **배경**: 엑셀 데이터 수정 후 수동 보정 단계에서 발생하는 데이터 불일치 및 회귀 오류 해결 필요.
- **결정**: 동기화 시 엑셀 원본 데이터가 존재할 경우 기존 DB 필드를 무조건 덮어쓰는(Overwrite) 정책 채택. (소급 적용은 금일 이후 건부터 적용)

## 6. 컴포넌트 간 'One-Step UI' 연동 정책 (2026-04-14)
- **배경**: 대시보드와 측정일지 간의 이동 빈도가 높으나, 매번 수동 검색과 클릭 과정이 반복되어 효율성 저하.
- **결정**: URL 파라미터(`code`, `autoOpen` 등)를 이용한 컴포넌트 간 직접 연동 도입. 검색 결과가 1건일 경우 즉시 상세 모달을 띄워주는 자동화 패턴을 표준화하여 향후 영업/정산 등 다른 컴포넌트 연동 시에도 확장 가능하도록 설계.
