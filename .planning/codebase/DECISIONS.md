# Design Decisions - 측정일지 관리 시스템

프로젝트의 방향을 좌우한 주요 설계 결정 사항들을 기록합니다.

## 1. Next.js 14 & Supabase 도입
- **이유**: 기존 엑셀 기반 관리 체계에서 다수 사용자가 동시 접속 가능한 웹 시스템으로 전환하기 위해 도입.
- **결과**: Auth, DB, API 구축 속도가 획기적으로 향상됨.

## 2. Selenium 기반 로우 레벨 자동화 선택
- **이유**: K2B 사이트의 복잡한 보안 및 동적 UI 구조로 인해 단순 API 연동이 불가능함.
- **결과**: 관리자의 수동 입력 시간을 대폭 단축하는 자동화 엔진 구축.

## 3. Direct Excel Stream Parsing
- **내용**: 엑셀 데이터를 CSV로 변환하지 않고 `xlsx` 라이브러리를 통해 직접 파싱함.
- **이유**: XLSX 파일의 다중 시트 및 서식 정보를 유지하면서 데이터를 직접 추출하기 위함.

## 4. 5인 미만 연번 번호 유지 전략
- **내용**: 5인 미만 사업장은 연번을 올리지 않고 직전 번호를 그대로 사용하여 누적 관리를 단순화함.
- **이유**: 비즈니스 운영 상의 편의성과 기존 관습 유지.
